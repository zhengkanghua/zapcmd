## 一、发版前检查清单

在执行任何操作之前，确认以下事项全部就绪：

### 1.1 代码状态

- [ ] 所有 CI/CD 修复已合入（参照 `01-ci-cd-fix.md`）
- [ ] `npm run check:all` 本地全绿
- [ ] `npm run tauri:dev` 可正常启动
- [ ] 功能手动验证无误（搜索、暂存、执行、设置页）

### 1.2 版本号同步

当前版本 `0.1.3`，需要升级到 `1.0.0`，涉及 3 个文件（由 `sync-version.mjs` 自动处理 2 个）：

| 文件 | 字段 | 当前值 | 目标值 |
|------|------|--------|--------|
| `package.json` | `version` | `0.1.3` | `1.0.0` |
| `src-tauri/Cargo.toml` | `version` | `0.1.3` | `1.0.0`（sync 自动） |
| `src-tauri/tauri.conf.json` | `version` | `0.1.3` | `1.0.0`（sync 自动） |

### 1.3 Tauri 签名密钥（必须完成）

updater 功能需要签名密钥对，当前 `tauri.conf.json` 中 pubkey 还是占位符。

## 二、详细操作步骤

### 第 1 步：生成 Tauri 签名密钥对

```bash
# 在项目根目录执行
npx @tauri-apps/cli signer generate -w ~/.tauri/zapcmd.key
```

执行后会：
- 提示输入密码（用于加密私钥），**务必记住这个密码**
- 生成两个内容：
  - **私钥文件**：`~/.tauri/zapcmd.key`
  - **公钥**：输出到终端（一串 Base64 字符串）

**立即记录**：
1. 复制终端输出的**公钥字符串**（以 `dW50` 开头的 Base64）
2. 记住输入的**密码**

### 第 2 步：配置公钥到 tauri.conf.json

打开 `src-tauri/tauri.conf.json`，替换 pubkey 占位符：

```json
"plugins": {
  "updater": {
    "endpoints": [
      "https://github.com/zhengkanghua/zapcmd/releases/latest/download/latest.json"
    ],
    "pubkey": "在这里粘贴第 1 步生成的公钥字符串"
  }
}
```

### 第 3 步：修改版本号

```bash
# 修改 package.json 中 version 为 1.0.0
# 然后运行 sync 脚本同步到 Cargo.toml 和 tauri.conf.json
npm run version:sync
```

手动验证三个文件都已更新为 `1.0.0`：

```bash
# 检查 package.json
node -e "console.log(JSON.parse(require('fs').readFileSync('package.json','utf8')).version)"

# 检查 Cargo.toml
findstr "version" src-tauri\Cargo.toml

# 检查 tauri.conf.json
node -e "console.log(JSON.parse(require('fs').readFileSync('src-tauri/tauri.conf.json','utf8')).version)"
```

三个都应输出 `1.0.0`。

### 第 7 步：将所有 commit 合并为一个（关键步骤）

**目标**：当前 main 分支上有 4 个 commit，我们要将它们压缩为 1 个干净的 commit。

**方法：Soft Reset + 重新 Commit**

```bash
# 1. 确认当前状态干净（无未提交改动）
git status

# 2. 查看当前有多少 commit
git log --oneline
# 预期看到类似：
# a4938ee 文档整理
# 742e015 检测更新 & 开机自启 & 关于页面
# 2c25643 delete github back
# d05bbcd Initial release: ZapCmd v0.2.0

# 3. 注意！在步骤 3-6 的改动还未 commit 的情况下，先 commit 这些改动
git add -A
git commit -m "chore: prepare v1.0.0 release"

# 4. Soft reset 到第一个 commit 的父节点（即清空所有 commit 但保留文件）
#    因为 d05bbcd 是第一个 commit（没有父节点），我们使用另一种方式：
git reset --soft $(git rev-list --max-parents=0 HEAD)

# 5. 这时所有改动都在暂存区，重新提交为一个 commit
#    先取消暂存再重新添加（确保干净）
git reset
git add -A
git commit -m "Initial release: ZapCmd v1.0.0

Desktop command launcher focused on speed, safety baseline, and repeatable workflows.

Features:
- Fast search with token-AND matching across 209 builtin commands
- Parameterized execution with UI argument fill
- Staging queue for batch command execution
- Safety baseline: dangerous command confirmation + arg injection guard
- Settings: hotkeys, default terminal, i18n (zh-CN/en-US), auto-update
- User custom commands (~/.zapcmd/commands/*.json)
- Cross-platform: Windows x64 / macOS arm64 / Linux x64"

# 6. 验证：只有一个 commit
git log --oneline
# 预期输出类似：
# abc1234 Initial release: ZapCmd v1.0.0
```

> **注意**：如果第 4 步的 `git rev-list --max-parents=0 HEAD` 命令在 Windows 上有问题，
> 可以改用以下等效方式：
>
> ```bash
> # 获取第一个 commit 的 hash
> git rev-list --max-parents=0 HEAD
> # 假设输出 d05bbcd...
>
> # 然后 soft reset（注意是 reset 到该 commit，不是它的父节点）
> # 由于这是 root commit，我们需要换一种方式：
>
> # 方法 B：直接删除 .git 重建（最干净）
> # 先确保所有文件在工作区
> rmdir /s /q .git
> git init
> git add -A
> git commit -m "Initial release: ZapCmd v1.0.0
>
> ..."
> git branch -M main
> ```

### 第 8 步：配置 GitHub Secrets

到 GitHub 仓库 Settings 配置签名密钥：

```
1. 打开 https://github.com/zhengkanghua/zapcmd/settings/secrets/actions
2. 点击 "New repository secret"
3. 添加两个 Secret：

   Name: TAURI_SIGNING_PRIVATE_KEY
   Value: （第 1 步生成的私钥文件的完整内容）
          打开 ~/.tauri/zapcmd.key，复制全部内容粘贴

   Name: TAURI_SIGNING_PRIVATE_KEY_PASSWORD
   Value: （第 1 步设置的密码）
```

### 第 9 步：推送到 GitHub

```bash
# 1. 确认远程仓库配置（如果之前重建了 .git，需要重新添加 remote）
git remote -v

# 如果没有 origin，添加：
git remote add origin https://github.com/zhengkanghua/zapcmd.git

# 2. 强制推送（因为我们重写了历史）
git push -u origin main --force

# 3. 验证推送成功
# 打开 https://github.com/zhengkanghua/zapcmd
# 确认只有 1 个 commit
# 确认 CI Gate 开始运行
```

### 第 10 步：等待 CI 通过

```
1. 打开 https://github.com/zhengkanghua/zapcmd/actions
2. 等待 "CI Gate" workflow 运行完成（绿色 ✓）
3. 如果失败，查看日志修复后重新推送
```

```
1. 打开 https://github.com/zhengkanghua/zapcmd/settings
2. 滚动到底部 "Danger Zone"
3. 点击 "Change repository visibility"
4. 选择 "Make public"

```
1. 打开 https://github.com/zhengkanghua/zapcmd/settings/security_analysis

## 三、回滚方案

如果发版过程中出现问题：

### Tag 推送后发现问题

```bash
# 删除远程 tag
git push origin --delete v1.0.0

# 删除本地 tag
git tag -d v1.0.0

# 修复问题后重新走第 13 步
```

### Release 已发布但有严重 bug

```
1. 到 GitHub Releases 页面
2. 编辑 v1.0.0 Release
3. 勾选 "Set as a pre-release" 或直接删除
4. 修复后发 v1.0.1 补丁版本
```

```

  ① 生成 Tauri 签名密钥对

  ⑫ 开启 Code Scanning

  ⑭ 验证 Release 产物
  ⑮ 安装验证
```

## 五、注意事项

1. **签名密钥是核心安全资产**：`~/.tauri/zapcmd.key` 和密码务必安全存储，丢失后将无法发布更新包给已安装用户
2. **force push 前确认**：重写历史后 force push 会覆盖远程，确保本地代码是完整正确的
3. **转 Public 不可逆**：Free plan 下 Private → Public 后，如果要转回 Private 需要联系 GitHub Support
4. **发版后第一时间验证 updater**：安装 v1.0.0 后，updater 能否正确检测到版本是关键功能
