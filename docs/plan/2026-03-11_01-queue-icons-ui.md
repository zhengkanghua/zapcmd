# 队列与暂存区图标按钮 UI 微调（实现计划）

> 创建日期：2026-03-11  
> 状态：待审批  
> 优先级：P1  

---

## 1. 背景与目标

1. 背景：主窗口队列入口与暂存区（Review Overlay）按钮以文字为主，识别效率与信息密度一般。
2. 目标（用户价值）：改为小图标按钮 + 数量徽标，减少占位、提升可扫读与一致性，同时保持无障碍与键盘可达性。

## 2. 范围（In / Out）

### 2.1 In Scope

1. 主窗口搜索条右侧队列入口：始终显示队列图标，`count>0` 显示徽标（`99+` 截断仅用于可视）。
2. Review Overlay：关闭/复制/移除/清空按钮改为纯图标（保留 `aria-label` + `title`）。
3. Review Overlay 头部：将“队列 + 数量”改为紧凑“小 Tab”并置于热键提示前，同一行对齐（不加发光、非交互）。
4. 新增一套可复用的 launcher 图标（内联 SVG，`currentColor`）。

### 2.2 Out of Scope

1. 不变更队列逻辑（入队/执行/清空/热键语义）。
2. 不引入第三方 icon 依赖库。

## 3. 用户故事

> 作为重度键盘使用者，我希望快速识别队列入口与暂存区操作按钮，并在队列非空时一眼看到数量，这样我可以更快完成“入队→预览→复制/移除/清空→执行”的流程。

## 4. 交互与行为口径（Current vs Roadmap）

1. Current（本需求落地后立即生效的行为）：
   - 队列入口按钮空队列也可见；点击可打开队列面板并展示空态。
   - 图标按钮无文字但仍可通过悬浮提示与读屏理解含义。
2. Roadmap：无（本次不规划后续演进）。

## 5. 数据与配置（如涉及）

不涉及。

## 6. 验收标准（可验证）

1. 主窗口队列入口始终存在；`count` 对应的可视徽标与读屏数值正确（`>=100` 仅可视显示 `99+`，读屏仍为真实值）。
2. Review Overlay 头部：小 Tab 在左、热键提示在右；不换行成“两行标题”；Tab 非交互（不可 focus）。
3. “关闭/复制/移除/清空”均为纯图标按钮，且 `aria-label` 与 `title` 正确。
4. `npm run check:all` 全绿。

## 7. 测试计划

### 7.1 自动化测试（必须）

1. 更新现有回归测试中的队列数量读取方式：从 `.queue-summary-pill` 的 `aria-label` 解析 `count`（避免依赖可视文本）。
2. 重点覆盖：
   - 入队后队列数量正确变化（0→1→0 等）
   - 既有 Review Overlay 组件测试不因按钮文案变化而失败

### 7.2 人工回归（按需）

1. 打开 `http://127.0.0.1:5173/`：
   - 右侧队列入口：空队列只显示图标；入队后出现徽标数量；点击能打开 Review。
   - Review 内按钮：X/复制/移除/垃圾桶图标可点、hover 与 focus 清晰。

## 9. 风险与回滚

1. 风险点：纯图标按钮可能降低首次理解；通过 `title` 与 `aria-label` 缓解。
2. 回滚策略：若观感不达预期，可快速回滚到文字按钮（仅改动视图层与 CSS，逻辑不变）。
