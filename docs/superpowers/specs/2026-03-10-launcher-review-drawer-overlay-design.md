# ZapCmd｜Launcher 抽屉式 Review（面板内 2/3 覆盖）设计稿

日期：2026-03-10  
状态：已与需求方对齐（待实现）  

## 1. 背景与问题

当前队列 Review 的呈现出现“右侧独立抽屉/窗口变宽/视觉分离”的观感，导致与最初重构目标不一致（期望更接近 Raycast/Alfred：在同一张搜索面板内部、搜索框下方进行抽屉展示）。

## 2. 目标（What）

在**不改变窗口宽度**（最大宽度=搜索面板宽度）的前提下，实现“抽屉式 Review”：
- 抽屉仅出现在**搜索框下方**的内容区（结果抽屉区域），不覆盖搜索框。
- 抽屉从**右向左**滑入，覆盖内容区宽度的 **2/3**（暂定）。
- 左侧 **1/3** 仍能透过“轻遮罩”看到当前查询结果上下文（可见但不可交互）。
- 点击遮罩即可关闭抽屉；关闭后**焦点回到搜索输入框**。
- 动画更顺滑（抽屉感明显），并提供 `prefers-reduced-motion` 降级。

## 3. 非目标（Out of Scope）

- 不做“挤压式”（push）布局（即不缩窄/挤压结果列表宽度）。
- 不在本阶段解决“长命令在窄栏展示策略”的所有细节（因本方案不缩窄面板整体宽度）。
- 不调整窗口尺寸策略（本次仅讨论面板内部层级与动效，不改变窗口宽度口径）。

## 4. 关键约束（Must）

1. **窗口不变宽**：Review 打开不引入额外列，也不通过 `--review-width` 扩展布局。
2. **抽屉在面板内**：抽屉必须挂载在搜索面板（search-main）内部，属于同一张容器的内容区 overlay。
3. **三层结构**（从下到上）：
   - 结果层（搜索结果/empty/floor）
   - 遮罩层（轻遮罩 scrim）
   - 抽屉层（review drawer panel）
4. **关闭与回焦点**：点击遮罩关闭时，抽屉消失 + 遮罩消失 + 焦点回到搜索输入框。

## 5. 信息架构（IA）与布局

### 5.1 内容区定义

“内容区”指搜索框（search capsule）下方区域，即：
- 有结果时：现有结果抽屉区域（result drawer）
- 无结果且 Review 打开时：floor 占位区域（floor height protection）

抽屉只覆盖内容区，不覆盖 search capsule。

### 5.2 抽屉覆盖宽度

- 抽屉宽度：内容区宽度的 `2/3`（暂定），右对齐。
- 左侧保留：内容区宽度的 `1/3`，用于“可见上下文”。

### 5.3 抽屉内部布局（允许重排）

抽屉层内部可按实现调整，但必须满足：
- 顶部：标题（队列数量）+ 提示文案 + 关闭按钮
- 中部：队列列表（可滚动；拖拽排序保留）
- 底部：清空 / 执行全部按钮区

## 6. 交互契约（Behavior）

### 6.1 打开

触发方式保持现有语义（例如 queue summary pill / 热键切换），打开后：
- 结果层不可交互（inert/aria-hidden 或等价方案），但仍可见
- 焦点进入抽屉层（沿用既有 focus trap 语义）

### 6.2 关闭

关闭入口：
- 点击遮罩（scrim）
- 点击 Search Capsule（搜索输入区域；queue pill 除外，避免误关）
- 既有关闭按钮
- 既有键盘后退语义（如 Esc：先关 Review）

关闭后：
- 抽屉层卸载或进入 closing 状态动画
- 遮罩层消失
- **焦点回到搜索输入框**

### 6.3 可达性（A11y）

- 遮罩层应是可聚焦/可操作的关闭入口（button + `aria-label=close`）。
- 抽屉层应保留 `role="dialog"`（或等价语义）。`aria-modal` **不强制**（后续引入左右双抽屉并存时无法保证严格模态），需确保屏幕阅读器不会误读底层结果为可操作内容（配合 `inert/aria-hidden` 或等价方案）。

## 7. 视觉与动效（Motion）

### 7.1 动效目标

“抽屉感”更明确、更顺滑：
- 抽屉：从右滑入/滑出（`transform`）
- 遮罩：轻淡入/淡出（`opacity`）

### 7.2 时长与曲线

- 总时长：约 `200ms`（与现有动效基线一致，避免状态机与 CSS 脱频）
- easing：抽屉采用更贴近 drawer 的曲线（快出慢收），遮罩采用 ease-out/ease-in
- `prefers-reduced-motion: reduce`：禁用动画，直接显示/隐藏

## 8. 验收标准（UAT / Done）

1. Review 打开时，窗口宽度不变；视觉上不再出现“右侧独立卡片/分离露底”。
2. 抽屉只出现在搜索框下方内容区；搜索框始终可见且不被覆盖。
3. 抽屉从右向左覆盖内容区 `2/3`；左侧 `1/3` 仍能看到结果上下文（轻遮罩下可见）。
4. 点击遮罩关闭抽屉；关闭后焦点回到搜索输入框。
5. 动画顺滑且无明显卡顿；`prefers-reduced-motion` 下无动画。

## 9. 风险与后续演进

- 后续若要做“挤压式”（push），需要单独评估长命令/高密度结果的展示策略与断行/省略规则。
- 若未来需要更接近 Alfred 的“列表 + 右预览”分栏模式，应单开设计稿，避免与本次 overlay 抽屉混杂。
