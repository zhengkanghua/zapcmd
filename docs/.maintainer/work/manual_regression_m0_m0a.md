# M0/M0A 人工回归测试清单

> 目的：自动化回归已覆盖主逻辑后，保留必须人工验证的真实环境能力。  
> 建议频率：每次改动热键、窗口行为、终端链路后至少执行 1 轮。  
> 执行环境：`npm run tauri:dev` 启动的真实桌面应用（不要只用浏览器预览）。

## 1. 本轮自动化覆盖说明

1. 主窗口热键与队列交互：已自动化覆盖。
2. 设置窗口热键录制、冲突校验、终端下拉键盘导航：已自动化覆盖。
3. 命令管理页（启停开关、冲突标记、导入校验提示）关键交互：已自动化覆盖。
4. 失败链路与事件分支（执行失败、设置保存失败、storage/broadcast 同步、拖拽重排）：已自动化覆盖。
5. 安全链路（高危确认、注入拦截）与会话恢复：已自动化覆盖关键分支。
6. 仍需人工验证：系统终端真实行为、窗口管理真实行为、跨平台终端探测一致性。

## 2. 人工回归用例

### TC-MAN-01 启动与基础输入

1. 启动应用，确认主窗口可见。
2. 在搜索框输入任意关键词（如 `docker`）。
3. 清空输入。

预期：
1. 主窗口正常显示，搜索框可输入。
2. 有输入时出现结果抽屉。
3. 清空后结果抽屉关闭。

### TC-MAN-02 Escape 行为链路（真实窗口）

1. 搜索命令并触发参数弹层。
2. 按一次 `Esc`。
3. 搜索框有文本时再按 `Esc`。
4. 队列展开时再按 `Esc`。
5. 在主界面空态再按 `Esc`。

预期：
1. 第一次关闭参数弹层。
2. 第二次清空搜索文本。
3. 第三次收起队列。
4. 最后触发主窗口隐藏。

### TC-MAN-03 终端执行可见性（真实系统终端）

1. 选择一个无参数命令执行（`Enter`）。
2. 执行一个队列命令（`Ctrl+Enter`）。
3. 执行一个预期失败的命令（例如故意输错参数）。

预期：
1. 命令在系统终端执行，不在应用内展示输出面板。
2. 成功命令在终端可见输出。
3. 失败命令在终端可见错误信息，应用不崩溃。

### TC-MAN-04 设置窗口保存与持久化

1. 进入设置窗口，修改任意本地热键和默认终端。
2. 点击保存，关闭设置窗口后重新打开。
3. 再次启动应用。

预期：
1. 保存成功提示出现后自动消失。
2. 重开设置窗口后值保持。
3. 重启应用后值仍保持。

### TC-MAN-05 设置窗口下拉交互（鼠标+键盘）

1. 打开默认终端下拉。
2. 用 `ArrowUp/ArrowDown/Home/End` 导航，`Enter` 选择。
3. 重新打开下拉，点击下拉外区域。
4. 下拉打开时按 `Esc`，再按 `Esc`。

预期：
1. 焦点项可移动，`Enter` 可选中。
2. 点击外部区域关闭下拉。
3. 第一次 `Esc` 关闭下拉，第二次 `Esc` 关闭设置窗口。

### TC-MAN-06 主窗口拖拽与多显示器

1. 在可拖拽区域拖动窗口到新位置。
2. 关闭并重开应用。
3. 有多显示器时，切换显示布局或断开副屏后再唤起应用。

预期：
1. 窗口可拖拽且交互区不误触。
2. 重开后窗口位置恢复合理。
3. 不会恢复到屏幕外不可见区域。

### TC-MAN-07 跨平台终端探测一致性

1. 在目标平台打开设置页“通用”。
2. 检查终端下拉项是否与本机已安装终端一致。
3. macOS 环境重点检查 iTerm2 的有/无安装场景。

预期：
1. 下拉项与本机实际环境基本一致。
2. 不可用终端不应被误报为可用。

### TC-MAN-08 用户命令目录加载（重启生效）

1. 在终端运行应用：`npm run tauri:dev`。
2. 创建用户目录命令文件（Windows 示例）：
   - 路径：`$env:USERPROFILE\\.zapcmd\\commands\\custom.json`
   - 内容使用 schema 合法格式（至少包含 `commands` 数组和一条命令）。
3. 在主窗口搜索该自定义命令 `name`/`template` 关键词。
4. 保持应用运行，修改该 JSON 中命令名称或模板内容并保存。
5. 重启应用后再次搜索并执行。

预期：
1. 首次保存后可被搜索命中并可执行。
2. 修改后需要重启应用，重启后可命中新内容。
3. 非法 JSON 或 schema 不合法文件会被跳过，不影响其他命令。

### TC-MAN-09 高危命令执行确认

1. 搜索“解除端口占用”等高危命令（dangerous=true）。
2. 填写参数后提交执行。
3. 在高危确认弹层点击“取消”，再重复并点击“继续执行”。

预期：
1. 首次提交不会直接执行，先出现高危确认弹层并展示风险原因。
2. 点击“取消”后不触发终端执行。
3. 点击“继续执行”后才会触发终端执行。

### TC-MAN-10 参数注入拦截

1. 选择带参数命令，参数输入 `443; whoami` 或 `8080 && echo hacked`。
2. 提交执行。

预期：
1. 执行被阻断，不触发终端执行。
2. 主界面出现“执行已拦截”错误反馈。

### TC-MAN-11 跨会话队列恢复

1. 将 1-2 条命令加入暂存队列并保持展开状态。
2. 关闭应用后重新启动。

预期：
1. 暂存队列恢复到关闭前状态（命令与参数值存在）。
2. 若关闭前为展开状态，启动后队列面板保持展开。

### TC-MAN-12 命令管理页启停与冲突标记

1. 进入设置窗口“命令”页。
2. 找到一个内置命令，取消勾选“已启用”，点击保存。
3. 回到主窗口搜索该命令关键词。
4. 重新勾选启用并保存，再次搜索确认恢复。
5. 在 `~/.zapcmd/commands` 放置与内置同 id 的用户命令，重启应用后进入“命令”页查看标记。

预期：
1. 禁用后命令不会出现在主窗口搜索结果中。
2. 重新启用后命令恢复可搜索。
3. 被用户命令覆盖的内置命令显示“覆盖内置”标记。

### TC-MAN-13 命令导入校验提示

1. 在 `~/.zapcmd/commands` 新建一个非法 JSON 文件（例如 `{invalid`）。
2. 重启应用并进入设置窗口“命令”页。

预期：
1. 页面展示导入校验提示（JSON 解析失败）。
2. 非法文件不会影响其他合法命令的加载和执行。

### TC-MAN-14 命令管理筛选/排序/文件筛选

1. 进入设置窗口“命令”页。
2. 在搜索框输入关键词（例如 `docker`），观察列表收敛。
3. 切换来源筛选（仅内置/仅用户）和状态筛选（仅已禁用）。
4. 切换“按文件筛选”，选择某个来源文件。
5. 切换排序（按标题/按状态），并点击“重置筛选”。

预期：
1. 各筛选项可组合生效，命中数量与列表一致。
2. 文件筛选可只显示来自目标文件的命令。
3. 排序切换后列表顺序变化符合选择规则。
4. 重置筛选后恢复默认视图。
