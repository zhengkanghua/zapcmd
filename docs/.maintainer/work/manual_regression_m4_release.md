# M4 发布人工回归清单

> 目标：覆盖自动化无法完整替代的“安装包 + 真机终端 + 跨平台”场景。

## 1. 执行范围

1. Windows（必测）：安装包安装、启动、执行、设置持久化。
2. macOS（目标支持时必测）：安装后启动、终端调用、权限弹窗行为。
3. Linux（目标支持时必测）：安装后启动、终端调用、桌面环境兼容性。

## 2. 用例

### RC-M4-01 安装包可安装与启动

1. 使用发布产物安装应用。
2. 首次启动应用并唤起主窗口。

预期：
1. 安装完成且可正常启动。
2. 主窗口可见、可输入、无崩溃。

### RC-M4-02 命令搜索与分词高亮

1. 输入单词搜索（如 `docker`）。
2. 输入多词搜索（如 `inspect docker`）。

预期：
1. 单词搜索可返回结果。
2. 多词搜索按 AND 生效。
3. 高亮按分词逐词展示（顺序无关）。

### RC-M4-03 单命令与队列执行（系统终端）

1. 直接执行一条命令。
2. 暂存两条命令后批量执行。

预期：
1. 打开系统终端并执行命令。
2. 队列在同一终端顺序执行，输出可区分步骤。
3. 成功时队列清空，失败时队列保留。

### RC-M4-04 设置持久化与热键

1. 修改默认终端与一个本地热键并保存。
2. 重启应用后再次检查。

预期：
1. 修改值持久化生效。
2. 唤起热键可正常触发主窗口。

### RC-M4-05 用户命令目录加载

1. 在 `~/.zapcmd/commands` 放入合法 JSON 命令。
2. 重启应用并搜索该命令。
3. 再放入非法 JSON 文件并重启。

预期：
1. 合法命令可搜索并可执行。
2. 非法文件被跳过且有导入校验提示，不影响其他命令。

### RC-M4-06 安全链路

1. 执行高危命令，确认出现二次确认弹层。
2. 参数输入注入特征（如 `&&`、`;`）后提交。

预期：
1. 高危命令需确认后才执行。
2. 注入参数被拦截，不触发终端执行。
