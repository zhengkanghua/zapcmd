# ZapCmd UI 设计规格（Current Implementation）

> 状态：Active
> 基线日期：2026-02-23
> 说明：本文描述当前代码已实现行为，不包含未落地目标设计。

---

## 1. 范围

当前实现覆盖：

1. 主唤起窗口（搜索 + 结果抽屉 + 暂存侧栏 + 参数弹层）
2. 设置窗口（快捷键、默认终端、命令管理、外观占位）

当前未覆盖：

1. 主题系统（外观页仅占位）

---

## 2. 布局与尺寸（代码常量）

- 主搜索区基线宽度：`680`
- 主搜索区最小宽度：`420`
- 暂存侧栏展开宽度：`300`
- 暂存侧栏折叠宽度：`72`
- 搜索区与侧栏间距：`12`
- 主窗口最小基线高度：`124`
- 结果项高度：`72`
- 结果抽屉默认最多行：`8`
- 暂存卡片估算高度：`96`

样式来源：`src/styles.css`

---

## 3. 搜索与结果

1. 搜索输入为实时过滤。
2. 过滤字段：`title + description + preview`（大小写不敏感子串匹配）。
3. 支持空格分词 AND：输入多个词时，需全部命中上述字段才会入选。
4. 返回最多 8 条结果。
5. 高亮逻辑为分词高亮：对输入按空格分词后，逐词高亮命中的文本片段（顺序无关）。
6. 空输入不显示结果抽屉。
7. 执行反馈以悬浮 toast 呈现，不占搜索区布局高度。
8. toast 默认约 3 秒自动消失。
9. 当前不支持 `@file`、`#category`、recent 等扩展语法。

---

## 4. 暂存侧栏

### 4.1 交互规则

1. 首次入队（0->1）自动展开。
2. 支持拖拽重排。
3. 支持队列内参数编辑并实时重渲染命令。
4. 支持单项移除与整队清空。

### 4.2 队列执行

1. 执行方式：按当前顺序串行执行，并在同一终端窗口中批量执行。
2. 成功：清空队列。
3. 失败：停止并保留队列。
4. 命中高风险策略（危险命令/管理员权限/高危规则）时，先进入执行确认弹层。
5. 确认弹层支持 `Enter` 确认、`Esc` 取消。

---

## 5. 参数输入

1. 参数在弹层中输入。
2. 必填参数为空时阻断提交。
3. 提交行为取决于进入弹层时模式：
   - `stage`：加入队列
   - `execute`：立即执行
4. 参数值命中注入防护规则（如 `;`、`&&`、`||`、换行、命令替换）时阻断执行。

---

## 6. 快捷键（默认值）

### 6.1 全局

- 唤起窗口：`Alt+V`

### 6.2 搜索区

- 显示/隐藏暂存区：`Tab`
- 切换焦点：`Ctrl+Tab`
- 上/下选择：`ArrowUp` / `ArrowDown`
- 执行当前命令：`Enter`
- 加入队列：`ArrowRight`
- 返回/隐藏窗口：`Escape`

### 6.3 队列区

- 执行队列：`Ctrl+Enter`
- 清空队列：`Ctrl+Backspace`
- 移除项：`Delete`
- 上移/下移：`Alt+ArrowUp` / `Alt+ArrowDown`

---

## 7. 窗口行为

1. 主窗口失焦自动隐藏（Rust 侧窗口事件处理）。
2. 主窗口位置持久化保存 `x/y`。
3. 设置窗口为独立窗口标签 `settings`。
4. 主窗口支持无边框透明壳层 + 自绘背景组件。

---

## 8. 设置窗口

### 8.1 Hotkeys

1. 可录制全局/本地快捷键。
2. 录制状态下，按 `Esc` 或点击录制按钮以外区域会取消录制。
3. 保存时校验空值与冲突。
4. 仅 `launcher` 会调用后端更新全局快捷键注册；其余保存在本地设置。

### 8.2 General

1. 检测可用终端（Tauri 后端）。
2. 支持设置默认终端。
3. 设置快照保存到 `localStorage` 的 `zapcmd.settings`（schema v1）。
4. 兼容读取历史 key：`zapcmd.settings.hotkeys`、`zapcmd.settings.general`（读取后迁移写回新 key）。

### 8.3 Commands

1. 提供命令管理列表，支持按命令粒度启用/禁用。
2. 列表展示命令来源（内置/用户）、用户命令覆盖内置标记（同 id 覆盖）。
3. 展示导入校验提示（非法 JSON / 非法 schema / 命令 id 冲突被跳过）。
4. 支持按关键词、来源、状态、覆盖状态、导入异常、来源文件筛选；支持默认/标题/分类/来源/状态排序。
5. 批量启用/禁用作用于当前筛选结果（不是全量命令集）。
6. 命令启停状态写入 `zapcmd.settings.commands.disabledCommandIds`，实时影响主窗口搜索结果集。
7. 支持展示方式切换：平铺列表 / 按来源文件分组（分组键为来源文件路径）。
8. 命令管理视图状态写入 `zapcmd.settings.commands.view`（筛选/排序/展示方式持久化）。
9. 展示方式入口为“图标切换组”（列表/分组），与“排序”同一行，减少筛选区占用空间。
10. 导入校验提示采用高可见告警样式（警示图标 + 红色边框/背景）。

### 8.4 Appearance

1. 当前为占位说明，未提供可配置项。

---

## 9. 当前限制

1. 命令输出只在系统终端查看，应用内不做输出浮层/输出面板。
2. 用户命令在应用启动时加载，修改后需重启应用生效（当前未接入文件系统事件驱动监听）。
3. 执行结果可见性依赖终端本身输出；应用内仅展示命令摘要级反馈。
4. 暂未引入组件测试与 E2E 测试。
5. 暂存队列支持跨会话恢复（`zapcmd.session.launcher`），当前仅恢复队列与面板展开态，不恢复焦点位置。
