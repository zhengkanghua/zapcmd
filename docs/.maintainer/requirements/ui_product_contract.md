# ZapCmd UI Product Contract（Current Baseline）

> 状态：Active（以当前代码实现为准）
> 基线日期：2026-02-23
> 实现来源：`src/App.vue` + `src/styles.css` + `src-tauri/src/lib.rs`

---

## 1. 当前不可变行为

1. 主唤起窗口为单窗口透明壳层，主视觉为搜索框。
2. 主界面包含三块：`Search Capsule`、`Result Drawer`、`Floating Staging`，其中暂存区可折叠为计数按钮。
3. 输入为空时不显示结果抽屉；输入非空时显示结果抽屉。
4. 搜索结果最多展示 8 条。
5. 默认全局唤起热键是 `Alt+V`（可在设置页修改）。
6. 结果区默认 `Enter` 为“直接执行当前命令”，不是“默认入队”。
7. 结果区“入队”操作使用 `ArrowRight` 或鼠标点击结果项。
8. 需要参数的命令先进入参数弹层；参数弹层提交后按触发来源决定“执行”或“入队”。
9. 队列执行入口在暂存区（按钮或快捷键 `Ctrl+Enter`）。
10. 队列执行成功后清空暂存区；失败后保留暂存区。
11. 主窗口失焦自动隐藏。
12. 命令输出统一在系统终端查看；应用内不提供输出浮层/输出面板（该能力已取消）。
13. 主窗口提供最小执行反馈文案（成功/失败），失败原因在主界面可见。
14. 执行反馈以 toast 方式展示并自动消失（约 3 秒）。
15. 对高风险命令（模板标记高危/需管理员权限或命中高危规则）执行前必须二次确认。
16. 参数包含明显注入符号（如 `;`、`&&`、`||`、换行、命令替换）时阻断执行并给出错误反馈。
17. 主窗口会恢复上次会话的暂存队列（命令与参数值）；若上次为展开状态则自动展开队列面板。

## 2. 界面结构

1. `Search Capsule`：输入框、键盘操作入口。
2. `Result Drawer`：结果列表、键盘提示、结果高亮。
3. `Floating Staging`：折叠按钮 + 暂存列表 + 执行/清空操作。
4. `Param Overlay`：参数填写弹层。
5. `Settings Window`：独立窗口（`hotkeys / general / appearance` 三个页签）。

## 3. 状态与显隐规则

### 3.1 暂存侧栏状态机

状态：`closed` / `opening` / `open` / `closing`

1. 暂存从 `0 -> 1` 时自动打开（`closed -> opening -> open`）。
2. 暂存已有内容时继续入队，保持当前开合状态，不强制切换。
3. 用户点击队列按钮可在展开/收起之间切换。
4. 清空暂存后不会自动关闭侧栏状态；只会让列表变空。

### 3.2 Esc 行为（主窗口）

1. 参数弹层打开：关闭参数弹层。
2. 高风险确认弹层打开：取消本次执行确认。
3. 查询非空：清空查询。
4. 侧栏展开：收起侧栏。
5. 以上都不满足：隐藏主窗口。

### 3.3 命令管理行为（设置窗口）

1. 命令管理页支持按关键词、来源、启用状态、覆盖状态、导入异常、来源文件进行筛选。
2. 命令管理页支持排序（默认/标题/分类/来源/状态）。
3. 命令管理页支持展示方式切换：平铺列表 / 按来源文件分组。
4. 批量启用/禁用仅作用于“当前筛选结果”。
5. 命令管理视图状态（筛选/排序/展示方式）会持久化到 `zapcmd.settings.commands.view`。
6. 导入校验提示采用告警样式展示（警示图标 + 高对比提示区），便于快速发现配置错误。

### 3.4 热键录制行为（设置窗口）

1. 进入热键录制后，按 `Esc` 可取消录制。
2. 进入热键录制后，点击录制按钮以外区域会取消录制。

## 4. 当前默认快捷键

- 全局：
  - 唤起窗口：`Alt+V`
- 搜索区：
  - 显示/隐藏暂存区：`Tab`
  - 切换焦点：`Ctrl+Tab`
  - 上/下选择：`ArrowUp` / `ArrowDown`
  - 执行当前命令：`Enter`
  - 加入队列：`ArrowRight`
  - 返回/隐藏窗口：`Escape`
- 队列区：
  - 执行队列：`Ctrl+Enter`
  - 清空队列：`Ctrl+Backspace`
  - 移除队列项：`Delete`
  - 队列上/下移：`Alt+ArrowUp` / `Alt+ArrowDown`

---

## 5. 执行语义

1. 单条执行：通过结果区 `Enter` 或参数弹层“立即执行”。
2. 队列执行：按暂存顺序串行执行。
3. 终端执行通过 Tauri 命令 `run_command_in_terminal` 完成。
4. 批量队列执行会在同一终端窗口内顺序执行，不再为每条命令单独打开终端。
5. 命令输出由终端承载（stdout/stderr 在终端中查看），应用内不做输出展示。
6. 应用内仅显示状态级反馈（例如“已发送到终端（含命令摘要）”“执行失败：xxx”），不承载日志输出内容。
7. 队列执行前若命中高风险策略，会弹出确认框并展示每条风险命令的原因与摘要。
8. 参数校验不通过或命中注入防护规则时，执行会被阻断且不会发起终端调用。
