## 一、命名

### 1.1 名称

**ZapCmd**

### 1.2 命名由来

- **Zap（闪电）** 象征速度、瞬时执行和高效能量释放
- 闪电代表即时响应——输入命令，瞬间执行，与"命令工具箱"快速高效的产品定位完美契合
- 3 个字母，极简，打字飞快，适合作为 CLI 命令前缀
- 全球开发者工具领域无同名项目，品牌辨识度高

### 1.3 命名理念

| 维度 | 说明 |
|------|------|
| **核心概念** | 闪电象征速度精神——瞬时、高效、强大 |
| **语义映射** | 闪电 = 即时执行（用户输入命令，ZapCmd 瞬间响应，如同闪电划破天际） |
| **工程实用** | 3 字母，CLI 友好，可作为命令前缀（`zapcmd`）、配置目录（`~/.zapcmd/`） |
| **品牌传播** | 有具体视觉形象（闪电 ⚡），利于社区传播（类比 Linux 企鹅、Docker 鲸鱼） |
| **全球适用** | 发音简单直观，英语/中文用户均可轻松记忆和拼写 |

## 二、Logo 设计

### 2.1 设计概念：闪电 + 终端提示符

Logo 由两个核心元素组合而成：

1. **闪电符号 ⚡** — 代表 ZapCmd 的品牌形象（速度、能量、即时执行）
2. **终端提示符 `>_`** — 代表命令行（ZapCmd 的产品本质），巧妙地融入闪电图形中

两个元素的融合传达了产品的核心价值：**瞬时执行的命令行工具**。

### 2.2 设计理念

**为什么选择这个方案：**

1. **双重语义融合** — 闪电 ⚡ 与终端提示符 `>_` 结合，同时传达"速度"和"命令行工具"两个核心概念
2. **极简几何** — 闪电的锯齿形状简洁有力，在 16px favicon 尺寸下依然可辨认
3. **视觉冲击** — 闪电的动态形态带来能量感，黄色配色带来活力和警觉性
4. **动画潜力** — 闪电天然适合闪烁、充能、释放等动画效果，可作为 App 执行状态的视觉指示器
5. **色彩鲜明** — 使用 ZapCmd Yellow 和 ZapCmd Navy 形成强烈对比，适应深浅两种主题

### 2.3 Logo 构成

文件路径：
- `docs/img/logo.png`（应用图标源）
- `docs/img/brand.png`（品牌展示图）

**几何参数：**

| 元素 | 形状 | 颜色 | 说明 |
|------|------|------|------|
| 闪电主体 | 锯齿形闪电 ⚡ | ZapCmd Yellow `#F1C40F` | 向下延伸的闪电形状，带有能量感 |
| 终端提示符 `>` | 等宽字符或几何箭头 | ZapCmd Navy `#2C3E50` | 融入闪电顶部或中部 |
| 光标 `_` | 水平短线 | ZapCmd Yellow `#F1C40F` | 位于 `>` 右侧，可做闪烁动画 |

### 2.6 动画规范

```css
/* 终端光标闪烁动画 */
@keyframes zapcmd-cursor-blink {
  0%, 100% { opacity: 1; }
  50% { opacity: 0; }
}

.zapcmd-prompt-cursor {
  color: #F1C40F;
  animation: zapcmd-cursor-blink 0.8s ease-in-out infinite;
}

/* 闪电充能动画 */
@keyframes zapcmd-charge {
  0% {
    opacity: 0.5;
    transform: scale(0.95);
  }
  50% {
    opacity: 1;
    transform: scale(1.05);
    filter: brightness(1.3);
  }
  100% {
    opacity: 0.5;
    transform: scale(0.95);
  }
}

.zapcmd-logo-charging {
  animation: zapcmd-charge 1.2s ease-in-out infinite;
}

/* 闪电释放动画 */
@keyframes zapcmd-strike {
  0% {
    opacity: 0;
    transform: translateY(-20px);
  }
  30% {
    opacity: 1;
    transform: translateY(0);
  }
  100% {
    opacity: 0;
    transform: translateY(10px);
  }
}

.zapcmd-logo-strike {
  animation: zapcmd-strike 0.4s ease-out;
}
```

**状态动画扩展：**

| App 状态 | 闪电/光标行为 | 含义 |
|----------|--------------|------|
| 就绪（Idle） | `_` 正常闪烁（0.8s） | "我在等你输入" |
| 搜索中 | 闪电微弱脉动 | "正在思考" |
| 执行中 | 闪电充能发光 | "正在工作" |
| 执行成功 | 闪电快速闪烁 + `✓` | "搞定了" |
| 执行失败 | 闪电变红 + 抖动 | "出错了" |

## 四、品牌禁止用法

| 禁止行为 | 原因 |
|----------|------|
| 拉伸或压缩 Logo 比例 | 破坏视觉美感 |
| 使用非品牌色 | 降低辨识度 |
| Logo 旋转超过 5° | 破坏稳定感 |
| 在杂乱背景上使用彩色 Logo | 应使用单色变体 |
| 将闪电改为其他形状 | 破坏品牌识别性 |

## 六、品牌语言

### 6.1 品牌口号

**主口号：**
- "瞬时执行，一触即发" (Instant execution, lightning fast)
- "让命令服务于人，而不是人去记忆命令"

**辅助口号：**
- "快如闪电的命令工具箱"
- "输入即执行，无需等待"
- "ZapCmd it, done."

### 6.2 品牌个性

| 维度 | 描述 |
|------|------|
| **速度** | 快速响应，即时反馈，零延迟体验 |
| **效率** | 简化流程，减少步骤，提高生产力 |
| **能量** | 充满活力，积极主动，赋能开发者 |
| **可靠** | 稳定执行，准确无误，值得信赖 |

### 6.3 语气与风格

- **简洁直接** — 避免冗长描述，直击要点
- **积极主动** — 使用主动语态，强调行动和结果
- **友好专业** — 保持专业性的同时，语气轻松友好
- **技术导向** — 面向开发者，使用技术语言但不过度复杂
