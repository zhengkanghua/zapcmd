# 11. B4 Visual Spec

> 本文把 B4 方案从“结构概念”细化到可直接出 Demo / 写样式的视觉规格层。
> 它不是最终像素锁死稿，而是 **主窗口** 实现前的设计规格基线。

## 范围说明

### 本文覆盖
- launcher 主窗口
- Search / Review 视觉关系
- 长命令、参数、badge、按钮排版

### 本文不覆盖
- `settings` 视觉重构
- `settings` 页头、导航、footer 的升级方案
- `settings` 的任何并入式设计

`settings` 继续保持独立窗口，本轮不在范围内。

## 目标

本文回答：

1. 主窗口在 B4 下的尺寸、层级、间距、宽度如何定义
2. Search / Review 两层的视觉关系如何建立
3. 长命令、参数、badge、按钮如何排版

## 一、主窗口视觉规格

## 1. 外层窗口感知

### 视觉目标
- 像一个高质量桌面工具面板
- 不是“黑底半透明网页卡片”
- 不是“毛玻璃悬浮玩具”

### 外层规则
- 保持圆角 shell
- 透明只做轻透，不做看穿
- 壁纸只能提供轻度氛围，不得抢主视觉

## 2. 顶部拖拽区

### 规格
- 高度：`18px`
- 位置：搜索 shell 最上方
- 不计入内容区高度
- Review 打开时仍保留

### 视觉要求
- 不需要显性视觉装饰
- 只做非常轻的透明拖拽带
- 不允许被 overlay 遮罩吞掉

## 3. 搜索输入区

### 规格
- 搜索输入高度：`48px`
- 搜索主字号：`18px`
- 输入区域左右内边距：`16px`
- 搜索区总内边距：`16px`

### 视觉要求
- 搜索框必须是第一视觉焦点
- 不做明显外描边，聚焦时采用品牌色柔和高亮
- 占位符比正文轻一档

## 4. 搜索结果抽屉

### 结果项规格
- 单条结果高度：`72px`
- 标题字号：`14px`
- 预览字号：`12px`
- meta badge 字号：`10px`
- 结果项圆角：`12px`
- 结果项左右内边距：`14px`

### 抽屉规格
- 抽屉顶部 hint 区：`22px`
- 抽屉 chrome：`12px`
- `drawerFloorHeight = 322px`

### filler 规则
- 当结果区不足 4 条高度时，抽屉容器补足到 `322px`
- filler 只承担视觉延展，不承担真实数据语义
- filler 建议做成：
  - 低对比表面层
  - 轻微渐隐到背景
  - 不要画出假的结果卡片边框

## 5. Queue Summary Pill

### 作用
- 在搜索态提示“队列存在”
- 是进入 Review 的轻量入口

### 推荐规格
- 高度：`28px ~ 32px`
- 圆角：`999px`
- 左右 padding：`10px ~ 12px`
- 字号：`12px`

### 视觉要求
- 要明显，但不能抢搜索输入框风头
- 使用品牌软底 + 细边框
- 可带数量，如：`3 queued`

## 6. Review Overlay

## 宽度
推荐起始宽度范围：
- `420px ~ 480px`

理由：
- 当前右栏 `300px` 对长命令明显不够
- `420+` 才足以让命令摘要、参数和操作按钮不挤在一起

## 最小可视高度
- 内容区最小高度：`322px`
- 内部列表允许滚动

## 外观
- 使用 `bg.surface-elevated`
- 边框比主搜索壳层稍强
- 阴影比主搜索壳层更明显，但不能浮夸

## 动画
- 右滑进入时长：`180ms ~ 220ms`
- 配合内部 shell dim 渐显
- 如果窗口先发生 resize，尽量先完成尺寸稳定，再开始 overlay 进入

## 7. 背景 dim / lock 层

### 作用
- 让用户知道“背景还在，但已经退到上下文层”

### 规格
- 只作用于内部 shell
- dim 建议：`rgba(11, 14, 20, 0.28 ~ 0.36)`
- 可选非常轻的 blur，但不建议过强

### 禁忌
- 不允许整窗蒙层
- 不允许遮掉圆角边界和原有 shell 轮廓

## 二、Review 内部组件规格

## 1. Review Header

### 内容
- 标题：例如 `队列 3`
- 简短说明：如“审阅、排序并执行已暂存命令”
- 关闭按钮

### 规格
- 标题字号：`15px`
- 说明字号：`12px`
- header 高度建议：`56px ~ 64px`
- 左右 padding：`16px`

## 2. Review Card

### 卡片目标
- 让长命令“摘要可读”，而不是“代码堆砌”

### 卡片信息层级
1. 标题（用户意图）
2. 命令摘要（截断后的预览）
3. 参数区 / 参数 badge
4. 底部操作：删除、排序提示、执行关系说明

### 推荐规格
- 卡片最小高度：`88px ~ 96px`
- 卡片内边距：`12px`
- 卡片圆角：`12px`
- 卡片间距：`8px`

### 命令摘要规则
- 默认最多两行
- 使用等宽字体
- 超出后省略
- 完整命令可在展开态或 title tooltip 中查看

### 参数区规则
- 参数较少时：直接 inline field
- 参数较多时：可折叠/换行
- 不要让参数输入把卡片撑成很重的表单

## 3. Review Footer

### 内容
- 清空
- 执行全部
- 可选关闭按钮（若 header 已有关闭，也可不重复）

### 规格
- sticky footer
- 高度建议：`52px ~ 60px`
- 与 header 风格呼应

## 三、颜色落位规则

## 品牌色落位
- 搜索输入 focus 态
- 结果选中态
- Queue Summary Pill
- Review 打开态的重要激活标识
- 主按钮（若是“主要动作”）

## 成功色落位
- 执行成功提示
- 启用状态
- 成功反馈 badge

## 危险色落位
- 删除
- 高风险提示
- 错误反馈

## 明确禁止
- 不允许品牌色继续用绿色
- 不允许结果选中和成功态继续共享同一视觉语言

## 四、动效与交互视觉

## 1. 结果项
- hover：轻微表面变亮
- active：品牌软底 + 左侧强调线
- focus-visible：清晰但不刺眼

## 2. Review
- dim 先出现，Review 再滑入
- 关闭时先滑出 Review，再移除 dim

## 五、视觉验收清单

- [ ] 搜索输入框是第一视觉焦点
- [ ] 结果区在 1 条结果时也不会显得塌陷
- [ ] Review 打开后，长命令明显更可读
- [ ] Queue Summary Pill 明显但不喧宾夺主
- [ ] 品牌色与成功色彻底分离
- [ ] 整窗边界没有因遮罩而暴露
- [ ] 拖拽区仍然保留并自然存在
