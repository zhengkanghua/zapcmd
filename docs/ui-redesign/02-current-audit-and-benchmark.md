# 02. Current Audit and Benchmark

## 当前项目技术栈

ZapCmd 当前前端/桌面栈：

- `Vue 3`
- `TypeScript`
- `Pinia`
- `Vite`
- `Tauri 2`
- `Rust`

仓库依据：

- `package.json`
- `src-tauri/Cargo.toml`

## 竞品公开栈信息（按本次可核验口径）

### Raycast
- 官方明确表述自己是 **native macOS app**。
- 官方扩展架构文章明确提到扩展运行依赖 `TypeScript/JavaScript`、`Node.js`、`Go` 等运行时层。
- 结论：**核心原生 + 扩展生态使用 Web/Node 技术**。

### Alfred
- 官方长期公开口径是 **native macOS app**，强调轻量和系统级效率。
- 官方未在本次核验材料中公开完整语言栈细节。
- 结论：**典型 native-first 路线**。

### Linear
- 本次可稳定核验到的公开口径，以官方招聘与官方开发者文档为主。
- 能较稳确认的是它长期深耕 `TypeScript` 生态，前端公开资料长期围绕 `React` 路线。
- 结论：**现代 Web 技术栈可以做出极高完成度的专业产品**。

## 关于“我们的栈是否吃亏”的结论

结论非常明确：**不吃亏。**

- 如果跟 `Raycast / Alfred` 比“系统级原生感”，Tauri + WebView 先天不如纯 native 路线，这是事实。
- 但如果目标是做一个 **跨平台、轻量、专业、商业感强的桌面启动器**，你们现在的栈完全够用。
- 当前观感不够成熟，主要不是因为 `Vue + Tauri`，而是因为：
  - 主界面结构没有真正收敛为单焦点
  - 视觉令牌不成体系
  - 暂存区与长命令展示策略不成熟
  - Settings 的 IA 与视觉层级过薄

所以后续应该优先投资源到 **产品结构 + 视觉系统**，而不是怀疑栈。

## 当前主窗口审计

参考：`docs/temporary_img/主窗口.png`

### 做得对的地方
- 搜索入口明确，第一眼知道这是命令启动器
- 结果列表可扫描性还不错
- 参数化命令已具备区分能力
- 队列能力已经被明确可视化，不是完全隐藏功能

### 当前主要问题

#### 1. 壁纸存在感过强
- 背景透出太多，主界面不够“稳”
- 用户看到的是“彩色壁纸上的半透明黑框”，不是一个成熟产品界面

#### 2. 双焦点竞争
- 左边是搜索与结果
- 右边是队列与执行
- 这两个区域在视觉上接近同级，用户会在“继续搜”还是“看队列”之间来回切

#### 3. 右侧窄栏不适合长命令
- 你们的命令不少都比较长，还有参数
- 窄栏显示完整命令非常吃亏
- 结果就是：要么截断过多，要么排版显得脏乱

#### 4. 绿色使用过度
- 选中态、激活态、按钮、强调线同时用绿
- 导致产品缺少品牌层次，只剩“工具感”

## 当前 Settings 审计

参考：`docs/temporary_img/settings.png`

### 做得对的地方
- 左侧导航 + 右侧内容的基础结构是合理的
- 表单分组逻辑基本通顺
- 底部操作区（取消 / 应用 / 确定）符合桌面用户预期

### 当前主要问题

#### 1. 更像后台配置页，不像产品设置中心
- 缺少页头区域
- 缺少“当前页是什么、为什么改这里”的上下文说明
- 缺少卡片化层级，信息显得铺平

#### 2. 图标系统过旧
- 当前导航图标表达偏弱
- 若继续使用字符/简化符号，会进一步拉低商业感

#### 3. 纯黑内容区太硬
- 标题栏是系统白
- 内容区是纯黑
- 中间缺少过渡层和产品表面层级

#### 4. 输入控件节奏单一
- 所有项几乎都是同一种视觉密度
- 缺少主次字段、描述文案、状态反馈的轻重关系

## 商业化产品常见做法（形态观察）

### Raycast 风格观察
- 根搜索始终单焦点
- 次级动作在 Action Panel 或子层中呈现
- 不会在主命令面板里长期并列两个同等级滚动区

### Alfred 风格观察
- 搜索结果区始终是主舞台
- 批处理能力存在，但不会与主结果并列争夺注意力

### Linear 风格观察
- 极度克制的信息密度
- 详情和次级信息通过 Peek / Review 进入
- 很少把“两个大工作区”同时长时间并列

## 设计基线判断

如果以当前截图做评分：

- 合理性：`7.5 / 10`
- 现代感：`6.5 / 10`
- 精美度：`5.5 / 10`

问题不是“丑到不能用”，而是 **产品形态已经到了必须统一系统语言的阶段**。
