## 一、问题诊断

### 1.1 CodeQL 报错根因分析

**现象**：CodeQL workflow 运行时出现多个错误和警告

**根因**：仓库是 **Private** 仓库，GitHub 对 Private 仓库的 CodeQL 有以下限制：

| 报错信息 | 根因 | 严重程度 |
|---------|------|---------|
| `Resource not accessible by integration` | Private 仓库缺少 GitHub Advanced Security 许可证，CodeQL API 不可访问 | 阻塞 |
| `Code scanning is not enabled for this repository` | Private 仓库未开启 Code Scanning（需要 Advanced Security） | 阻塞 |
| `CodeQL Action v3 will be deprecated in December 2026` | v3 即将废弃，建议升级到 v4 | 低 |
| `Feature flags do not specify a default CLI version` | 非关键，使用 Action 内置的 CLI 版本即可 | 忽略 |

**结论**：Private 仓库的 Free plan 不支持 CodeQL code scanning。有两条路：
1. 发版 1.0.0 时转为 Public 仓库 → CodeQL 自动生效（**推荐，与发版计划对齐**）
2. 或者购买 GitHub Advanced Security for Private repos（不推荐，成本高）

### 1.2 Commit 自动化测试丢失根因分析

**现象**：改名之后，commit 时不再自动执行测试

**排查路径**：

```
pre-commit hook 链路：
  .githooks/pre-commit
    → npm run precommit:guard
      → scripts/precommit-guard.mjs
        → lint → typecheck → test:related → typecheck:test → cargo check
```

**已确认**：
- `git config core.hooksPath` 输出为 `.githooks` ✅ hooks 路径配置正确
- `.githooks/pre-commit` 文件存在 ✅
- `scripts/precommit-guard.mjs` 逻辑正确 ✅
- `scripts/setup-githooks.mjs` 在 `npm install` 时通过 `prepare` 脚本自动设置 hooks 路径 ✅

**可能的根因**：
1. 改名过程中执行了 `git init` 重建历史（见 rename plan 第三步），这会重置 `core.hooksPath` 配置
2. 但后续执行 `npm install` 或 `npm ci` 时，`prepare` 脚本应该会自动恢复
3. 如果改名后没有重新执行过 `npm install`，则 hooks 不会被自动重新设置

**当前状态**：`core.hooksPath = .githooks` 已经是正确的，说明 hooks 已恢复。如果你现在 commit 仍然不触发测试，问题可能是：
- Windows 上 `.githooks/pre-commit` 文件缺少执行权限
- 或者 Git for Windows 的 shell 无法正确执行 `#!/bin/sh` 脚本

### 1.3 GitHub Actions CI Gate 状态

**ci-gate.yml 触发条件**：

```yaml
on:
  pull_request:
  push:
```

这意味着**每次 push 和 PR 都会触发** CI。但 Private 仓库有以下限制：
- Free plan：每月 2000 分钟 Actions 时间（共享所有仓库）
- 如果分钟数用完，Actions 不会运行
- 需要在 Settings → Actions → General 中确认 Actions 已启用

## 二、修复方案

### 2.1 短期修复（发版前）

#### Fix 1：验证本地 pre-commit hook 可用性

```bash
# 1. 确认 hook 路径配置
git config core.hooksPath
# 预期输出: .githooks

# 2. 如果不是 .githooks，手动修复
git config core.hooksPath .githooks

# 3. 测试 hook 是否工作：做一个小改动然后 commit
# 如果 hook 执行了，会看到 lint/typecheck/test 输出
# 如果直接 commit 成功没有任何输出，说明 hook 没生效
```

#### Fix 2：Windows 执行权限问题修复

如果 hook 仍然不触发，可能是 Windows 上的 shell 兼容性问题：

```bash
# 检查 Git 使用的 shell
git config --get core.autocrlf

# 确保 .githooks/pre-commit 使用 LF 换行符（不是 CRLF）
# 可以用 VS Code 打开，右下角查看行尾序列
```

#### Fix 3：CodeQL 临时处理

在仓库仍为 Private 期间，有两个选择：

**选项 A：暂时禁用 CodeQL（推荐）**

修改 `.github/workflows/codeql.yml`，添加条件使其仅在 public 仓库运行：

```yaml
# 在 jobs.analyze 中添加条件
jobs:
  analyze:
    # CodeQL requires GitHub Advanced Security for private repos
    # Skip when repo is private, will auto-enable when made public
    if: github.event.repository.visibility == 'public'
```

这样 Private 时不运行，转 Public 后自动恢复。

**选项 B：直接升级到 CodeQL v4 + 加条件**

```yaml
- name: Initialize CodeQL
  uses: github/codeql-action/init@v4    # v3 → v4
  with:
    languages: ${{ matrix.language }}

- name: Perform CodeQL Analysis
  uses: github/codeql-action/analyze@v4  # v3 → v4
  with:
    category: "/language:${{ matrix.language }}"
```

**推荐选项 B**，一步到位解决 v3 废弃警告 + 添加 visibility 条件。

### 2.2 长期方案（随 1.0.0 发版）

1. 仓库转为 Public → CodeQL 自动可用
2. 在 GitHub 仓库 Settings → Code security and analysis → 启用 "Code scanning"
3. CodeQL v4 升级已在上面完成
4. 添加 `security-events: write` 权限（已有）+ 确认 `actions: read` 权限

## 三、代码改动清单

### 3.1 文件改动

| # | 文件 | 改动内容 | 优先级 |
|---|------|---------|--------|
| 1 | `.github/workflows/codeql.yml` | CodeQL v3 → v4 + 添加 `if: visibility == 'public'` 条件 | P0 |
| 2 | `.github/workflows/ci-gate.yml` | 添加 `permissions` 显式声明（best practice） | P1 |
| 3 | `.githooks/pre-commit` | 确认 LF 换行符（不改内容） | P0 |

### 3.2 codeql.yml 完整改动

```yaml
name: CodeQL

on:
  push:
    branches:
      - main
  schedule:
    - cron: "20 3 * * 1"

permissions:
  contents: read
  security-events: write
  actions: read          # 新增：telemetry 需要

jobs:
  analyze:
    name: Analyze (CodeQL)
    # CodeQL code scanning 仅在 Public 仓库免费可用
    if: github.event.repository.visibility == 'public'
    runs-on: ubuntu-latest
    timeout-minutes: 60

    strategy:
      fail-fast: false
      matrix:
        language:
          - javascript-typescript

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Initialize CodeQL
        uses: github/codeql-action/init@v4      # v3 → v4
        with:
          languages: ${{ matrix.language }}

      - name: Install dependencies
        run: npm ci

      - name: Perform CodeQL Analysis
        uses: github/codeql-action/analyze@v4    # v3 → v4
        with:
          category: "/language:${{ matrix.language }}"
```

### 3.3 ci-gate.yml 添加 permissions

```yaml
name: CI Gate

on:
  pull_request:
  push:

permissions:
  contents: read       # 新增：显式最小权限

concurrency:
  group: ci-gate-${{ github.ref }}
  cancel-in-progress: true
# ... 其余不变
```
