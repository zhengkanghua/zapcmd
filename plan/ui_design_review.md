# ZapCmd UI 与交互设计审查报告

## 一、产品视角 (Product Perspective)
目前 ZapCmd 作为一个面向程序员的高效命令管理工具，核心操作流（搜索 -> 暂存 -> 批量运行）逻辑已基本跑通。**结合以 Raycast / Alfred 等顶级效率工具为视觉与交互标杆的产品定位**，从体验上来看：
1. **工作流认知升级**：目前的“暂存区 (Staging)”在感知上更像一个“购物车”。针对批量运行的场景，可以考虑在产品底层将其包装为**“脚本编排器 (Script Composer)”** 或 **“执行流 (Execution Flow)”**。这样不仅能够容纳目前的批量运行，还能自然衔接未来“自定义组合命令”的功能。
2. **快捷操作高亮展示**：虽然有 `keyboard-hint` 等快捷键提示，但视觉层级偏弱。作为效率工具，键盘操作是核心，建议引入类似 Mac 上 `<kbd>` 实体按键风格的微标签，让用户一眼扫过就能形成肌肉记忆。
3. **沉浸式空态与引导**：抽屉空态 (Empty State) 当前提示略显冷淡，可针对无搜索结果增加“去创建自定义命令”的快速入口（Quick Action），形成闭环。

## 二、设计视角 (Design & UX)
界面整体采用了现代化的深色模式与圆角（12px），但在质感和细节上基于真实截图有如下进阶优化空间：
1. **材质与空间感 (Material & Depth)**
   - **面板嵌套层级**：从截图中看到，暂存区（Review Overlay）是带圆角的面板悬浮在主列表上，并且有深色的全屏遮罩。虽然区分了前后景，但内外两个面板的圆角差异不明显，边缘融合较生硬。建议内部浮层（如 Review 面板）采用比外部容器更小一点的圆角，或者通过微微的内阴影（`box-shadow: inset 0 1px ...`）和外侧的高斯模糊投射，增强物理悬浮感。
   - **模糊反馈 (Glassmorphism)**：底部遮罩层显得过于浓重纯黑（半透明黑），削弱了底层内容的可见性。可以改为带有适度 Blur（如 `backdrop-filter: blur(8px)`）的较浅遮罩，保留背景隐约轮廓的高级感。
2. **色彩体系优化 (Color Palette)**
   - **高亮与状态色**：目前选中的列表项和暂存区左侧指示条的颜色偏低调（略显暗淡的冷灰蓝）。建议在保持克制的同时，适当提高品牌/高亮色的明度和饱和度（如更明亮的青色或电光蓝），使当前焦点状态（Focus/Active）能够第一眼跃出屏幕。
   - **删除与危险操作**：红色垃圾桶图标的背景板在深色模式下不够通透，建议改用红色半透明背景搭配亮红 Icon（如 `bg-red-500/10 text-red-400`），避免与周边深灰色块产生视觉粘连。
3. **排版与负空间 (Typography & Spacing)**
   - **命令预览字体**：主列表显示的命令（如 `cat ~/.ssh/config`）当前使用了普通 Sans-serif。在面向程序员的工具中，所有涉及实际代码的部分必须强制使用精美等宽字体（Monospace，如 JetBrains Mono 或 Fira Code），以提升与系统终端一致的专业感。
   - **排版密度与类 Raycast 体验**：对标 Raycast 和 Alfred，主界面的命令列表项行距较为宽松。作为高频极客工具，UI 应当极度紧凑。适当缩小列表纵向内边距（Padding-Y）和图标尺寸，让一屏内透出更多信息，给用户“掌控一切”的高效感。

## 三、前端工程师视角 (Frontend Tech)
1. **组件化与组合式 API**：项目结构清晰（拆分了 LauncherSearchPanel、Overlay 等）。通过 `useLauncherWatchers` 和 `useLauncherHitZones` 等提取逻辑是很好的实践，建议继续贯彻。
2. **动画实现机制优化**：在 `styles.css` 中有较多手写的 `@keyframes`，由于 Vue 内置了 `<Transition>`，建议将其封装为复用的无头过渡组件（如 `<SlideFadeTransition />` 或 `<ScaleTransition />`），并将魔法数字（如 200ms）在 Tailwind 配置或 CSS 变量中统一收口为 `--duration-fast` / `--duration-normal` 等。
3. **焦点陷阱 (Focus Trap) 与可访问性 (A11y)**：
   在打开暂存区（Review Overlay）等模态面板时，尽管现有的 inert 属性非常有用，但可以引入专门的 Focus Trap 机制（防止通过 Tab 键将焦点移出面板区域漏回下层元素），提升全键盘操控的安全感。

## ⭐️ 落地建议结论
1. **视觉改版第一步：空间与聚焦（对齐原生质感）**：弱化浓重深黑遮罩，引入有层次的高斯模糊机制；强化焦点元素的色彩明亮度和物理悬浮感，达到 Raycast 级别的操作系统级原生融合。
2. **排版升级第二步：紧凑与极客统一**：所有涉及指令、参数、路径的地方强制引入现代化等宽字体（Monospace），全面收缩列表的纵向 Padding，实现高密度信息展示，并优化快捷操作入口的图形徽标化。
3. **交互增强第三步（动效）**：将 CSS ease 过渡全面升级为贝塞尔弹性过渡 (Cubic-bezier Bounce)，提升卡片弹出与列表聚焦的呼吸感。
4. **产品延伸第四步：编排心智**：在 UI 上预先给“自定义组合脚本”流出专属视图空间，将“暂存区”更名为更接近“编排工作流”的命名（如 Flow 或 Sequence），增加编排与配置入口。

## 四、待办（可迭代 Backlog）

> 目标：把“审查建议”变成可执行的 UI 精修清单，便于逐条实现与回归验证。

### P0（先做，收益最大）

- [ ] **动效统一为 Token**：把 140/160/200/220/300ms 等散落值收口到 `src/styles.css` 的 motion 变量（例如 `--ui-motion-fast/normal`），避免“某个面板太快/太慢”。
- [ ] **遮罩体系统一**：整理 Launcher/Settings 的 scrim（颜色、透明度、blur），形成 1 组可复用的变量（例如 `--ui-scrim-bg` / `--ui-scrim-blur`），避免层级感不一致。
- [ ] **焦点可视化增强**：统一 focus ring（边框 + 外发光）与 active 状态色，确保深色背景下“一眼可见当前操作点”。
- [ ] **代码/路径一律等宽字体**：所有命令预览、参数值、路径片段强制 `var(--ui-font-mono)`，提升“极客感”和可读性。

### P1（细节质感）

- [ ] **面板层级更真实**：内外面板圆角做层级区分（内层略小/阴影略弱），并增加微弱内阴影避免边缘粘连。
- [ ] **密度调参**：主结果列表、暂存卡片、设置页表单的 padding/行高形成一组“紧凑/标准”档位，避免各区密度漂移。
- [ ] **按钮语义与危险色**：危险操作统一 `btn-danger`（浅红底 + 高饱和红字），中性按钮统一 muted 风格，避免误触与视觉噪音。

### P2（可达性与一致性）

- [ ] **Focus trap 策略收口**：Review/Flow/Settings 内置确认层的 Tab 范围策略统一（优先“当前面板内循环”）。
- [ ] **Reduce motion 覆盖完善**：所有新增/现有过渡在 `prefers-reduced-motion` 下都能关闭（包括 Settings 内置确认层）。

### P3（产品化增强，非阻断）

- [ ] **空态闭环**：无结果 / 队列为空 的空态加入“下一步操作”（如：创建命令入口/打开帮助），减少用户迷路。
- [ ] **快捷键提示组件化**：把 `<kbd>` 质感与对齐做成可复用组件（统一大小、边框、高亮色）。
