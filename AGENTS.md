# ZapCmd 会话启动规则（仓库根）

本文件用于 **Codex / Claude / 其他 Agent 在进入仓库根目录后的会话启动兜底**。

## 强制启动顺序（不可跳过）

在执行任何实质性工作前，必须按以下顺序读取：

1. `./CLAUDE.md`
2. `./.ai/AGENTS.md`
3. `./.ai/TOOL.md`

## 强制触发场景

以下场景必须重新确认上述 3 个文件已读取：

- 新会话刚进入本仓库时
- 首次响应本仓库内的用户请求时
- 启动任何子代理、规划代理、执行代理前

## 最低执行要求

- 全程使用中文回答
- 回答代码/仓库相关问题前，先做搜索定位，禁止猜测
- 若 `./.ai/AGENTS.md` 与其他文档冲突，以 `./.ai/AGENTS.md` 为高优先级项目规则
- 若 `./.ai/AGENTS.md` 要求遵循工具规范，必须继续读取 `./.ai/TOOL.md`
- 若需要给用户打开本地预览 / 可视化 companion / 开发服务地址，必须先确认当前 Agent 运行环境与用户实际打开环境是否一致（例如 WSL Linux vs Windows 宿主）
- 若运行环境与用户打开环境不一致，禁止直接提供 Agent 侧 `localhost/127.0.0.1`；必须改为用户环境可访问的地址（如 WSL IP、`wsl.localhost` 映射名或明确可达的转发地址）
- 若无法从当前上下文确定用户的打开环境，必须先询问，再生成地址

若未完成上述启动读取，不得开始分析、修改、生成计划或调用高成本工具。
