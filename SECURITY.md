# Security Policy

## Supported Scope

Security fixes are prioritized for:

1. Latest release tag.
2. Current active development branch.

## How to Report

Do not disclose vulnerabilities in public issues.

1. Use GitHub Security Advisory (preferred, private).
2. If unavailable, open issue title:
   `[SECURITY] private report request`
   and do not include vulnerability details.

## Include in Report

1. Affected version/tag.
2. Reproduction steps.
3. Impact.
4. Optional fix suggestion.

## Response SLA

1. Acknowledgement: within 3 business days.
2. Triage result: within 7 business days.
