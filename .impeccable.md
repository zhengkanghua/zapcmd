# ZapCmd Impeccable Context

## Design Context

### Users
ZapCmd 的主要用户是开发者、技术运维与高频使用命令行的高级用户。典型使用场景是快速检索命令、补全参数、暂存执行流并在低容错环境下完成批量执行。界面需要优先服务“高频、短停留、低误操作容忍度”的桌面工作流，而不是低密度的展示型页面。

### Brand Personality
整体产品气质应为“专业、果断、可控”。界面首先要传达可靠、稳定、可预期，让用户在执行潜在高风险命令时感到放心；但这不等于只能做保守、无个性的视觉。局部可以更鲜明、更有记忆点，前提是始终服务信息层级、状态表达和操作效率。情绪目标优先级为：信任感、专注感、效率感，其次才是愉悦感与识别度。

### Current Visual Baseline
当前默认基线仍可延续深色 `obsidian` 方向，例如温暖深灰背景、琥珀或高能强调色、清晰对比层级、低噪音的玻璃感或材质点缀。但这只是当前基线，不是唯一允许的视觉答案；后续允许按主题系统扩展出不同情绪和色温的方案，只要不破坏桌面产品的效率与可读性。

### Product UI Principles
1. 先扫一眼就懂：信息层级必须支持快速扫读，标题、状态、风险和主操作一眼可辨。
2. 键盘优先：所有核心流转都要尊重键盘路径、焦点语义和可达性，不让视觉装饰压过操作效率。
3. 风险可感知：危险动作、执行态、失败态和权限语义必须明确暴露，不隐藏失败，不弱化反馈。
4. 高密度但不拥挤：允许桌面工具所需的信息密度，但通过节奏、间距、对比和分组保持秩序。
5. 装饰必须有职责：颜色、发光、模糊、渐变、动效只能用来强化焦点、分组、状态、空间层次或反馈，不能只为“好看”抢占注意力。

### Theme Strategy
ZapCmd 需要支持多主题，而不是把 impeccable 理解成单一的深色风格说明书。当前的 `obsidian` 只是默认基线主题，未来可以扩展浅色、高对比、冷暖差异更明显或更具氛围感的主题，但都必须满足以下条件：

1. 继续复用 `--ui-*` token 与既有主题系统，不在组件里写死主题语义。
2. 保持主操作、危险操作、禁用态、执行态和反馈态的语义稳定，不因换主题而失真。
3. 保持高信息密度界面的可扫描性、可读性和键盘可达性，不把桌面工具做成展示页。

### Effects Policy
允许使用发光、渐变、柔和投影、模糊、纹理和有限动画，但它们必须是“受控效果”，并遵守以下规则：

1. 发光优先用于焦点、激活态、可交互热点、执行反馈或主题氛围，不应用于大面积长期干扰型装饰。
2. 模糊和玻璃感不能影响文字、图标和状态标签的清晰度，尤其不能削弱搜索、命令列表和风险提示的可读性。
3. 渐变和高能色可以出现，但应围绕局部层级或主题表达展开，避免整页霓虹化、廉价赛博化或模板化 SaaS 感。
4. 动效需要尊重 reduced motion，并避免在高频输入、键盘导航和危险操作附近制造视觉负担。

### Anti-Patterns
1. 不要把“专业”误解成“不能发光、不能有氛围、只能极度克制”。
2. 不要把当前 `obsidian` 基线误写成唯一主题，更不要在 skill 指令里固化成单一 mood board。
3. 不做营销官网式 hero、夸张叙事、低信息密度留白优先的桌面界面。
4. 不做整页赛博霓虹、无节制炫光、为了视觉冲击牺牲可读性和操作效率的方案。
