# Milestone v1.0: 质量门禁与回归基线

**Status:** ✅ SHIPPED 2026-03-06
**Phases:** 1-12
**Total Plans:** 35

## Overview

v1.0 以“功能改动必回归”为核心目标，建立了覆盖率、关键路径回归、Rust 高风险模块测试、UI/UX 小幅精修，以及 macOS desktop smoke 口径收敛的完整质量基线。最终交付把本地脚本、CI / Release workflow、公开文档与 planning 审计全部统一到同一现实，并保留后续 v2 的 deferred 目标。

## Milestone Summary

**Key Decisions:**
- 采用 `check:all` + coverage 90% 作为稳定合并门禁。
- 保持桌面端 E2E 为最小稳定基线，Windows 为唯一 blocking desktop gate。
- 对历史 evidence 采用 correction note 更正，而不是覆盖原始执行记录。

**Issues Resolved:**
- 覆盖率门禁未达标与关键路径回归不足。
- Rust 侧高风险模块长期缺乏单测覆盖。
- Phase 10 的 macOS local / CI / docs 口径漂移。

**Issues Deferred:**
- `E2E-02` full-matrix desktop E2E 继续 deferred 到后续 milestone / v2。
- `SYNC-01`、`SEC-02` 仍待后续需求定义。

**Technical Debt Incurred:**
- macOS desktop smoke 仍受 SafariDriver / WKWebView 稳定性影响，只保留 experimental / non-blocking probe。
- Windows Git hook 在当前环境存在 `sh.exe ... couldn't create signal pipe, Win32 error 5`，需要后续单独修复。
