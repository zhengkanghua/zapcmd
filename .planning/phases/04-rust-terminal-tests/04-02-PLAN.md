<task type="auto">
  <name>Task 1: 将终端探测抽离为“可注入 resolver + 纯解析函数”（保持行为不变）</name>
  <files>src-tauri/src/terminal.rs</files>
  <action>
目标：让单测不调用真实 `where/which`，也不依赖 runner 是否安装终端程序。

1) 抽离 stdout 解析为纯函数（跨平台可测）：
   - `parse_first_non_empty_line(raw: &str) -> Option<String>`
   - 规则：按行 split，`trim` 后取第一条非空；若没有则 None
   - `command_path()` 内部改为复用该函数（保持对外返回一致）

2) 为各平台新增“可注入依赖”的 resolver（函数签名可调整，但必须满足：不触发进程/文件系统 IO）：
   - Windows：
     - `resolve_windows_terminals(exists: impl Fn(&str)->bool, path: impl Fn(&str)->Option<String>) -> Vec<TerminalOption>`
   - Linux：
     - `resolve_linux_terminals(exists: impl Fn(&str)->bool, path: impl Fn(&str)->Option<String>) -> Vec<TerminalOption>`
   - macOS：
     - `resolve_macos_terminals(path_exists: impl Fn(&str)->bool) -> Vec<TerminalOption>`

3) `get_available_terminals()` 仍作为 tauri command，但内部改为：
   - 调用对应 resolver，并把真实依赖（`command_exists/command_path/Path::exists`）作为闭包传入
   - 保持现有行为：包含项、回退项、默认 id 与 label 文案不应变化（除非必要的最小重构）

4) 组织测试模块入口：
   - 在 `terminal.rs` 末尾新增：`#[cfg(test)] mod tests_discovery;`

2) 额外检查：单测不应触发 `which/where` 或 `Path::exists` 的真实依赖（通过 resolver 注入确保）

<task type="auto">
  <name>Task 2: 补齐终端探测单测（id 集合断言 + 回退 + where/which 解析）</name>
  <files>src-tauri/src/terminal/tests_discovery.rs</files>
  <action>
新增 `src-tauri/src/terminal/tests_discovery.rs`，覆盖以下内容：

1) `parse_first_non_empty_line()`：
   - 多行输出（含空行/空白行）能正确取第一条非空并 trim
   - 全空/全空白返回 None

2) Windows（`#[cfg(target_os = \"windows\")]`）：
   - 用 stub exists/path 驱动 `resolve_windows_terminals(...)`
   - 断言只锁定 terminal_id 集合与回退行为：
     - 当 `exists` 对 `powershell/pwsh/wt/cmd` 都返回 true：结果必须包含这 4 个 id
     - 当全部为 false：必须回退到 `powershell`（至少 1 项）
   - 对 path：不锁定具体值，但应保证非空（或当 stub 返回 Some 时应透传）

3) macOS（`#[cfg(target_os = \"macos\")]`）：
   - `resolve_macos_terminals(...)`：
     - 必须包含 `terminal`
     - 当 stub 表示 iTerm 路径存在：必须包含 `iterm2`
     - 当不存在：不应包含 `iterm2`

4) Linux（`#[cfg(all(unix, not(target_os = \"macos\")))]`）：
   - `resolve_linux_terminals(...)`：
     - 当 stub 指定某个终端存在（如 `gnome-terminal`）：必须包含对应 id
     - 当全部为 false：必须回退到 `x-terminal-emulator`

注意：
- 按 CONTEXT 决策：不锁定排序、label、具体 path（除 resolver 透传 stub 的最小校验外）
  </action>
