<task type="auto">
  <name>Task 1: 将终端执行逻辑拆为 “build 命令 + spawn 执行”（保持外部行为不变）</name>
  <files>src-tauri/src/terminal.rs</files>
  <action>
目标：让单测在不 `spawn()` 的情况下验证 Windows/macOS/Linux 的 argv/脚本契约；同时不改变 `tauri::command` 对外行为。

1) 保持对外入口不变：
   - `run_command_in_terminal(terminal_id: String, command: String) -> Result<(), String>` 仍然存在
   - 仍然 `trim()` 并拒绝全空白命令（拒绝信息可保持当前 `"Command cannot be empty."`，但后续测试不锁定文案）

2) 新增最小内部 helper（命名可调整，但职责必须分离）：
   - `sanitize_command(command: &str) -> Result<String, String>`：实现 `trim()` + 全空白拒绝；返回“可执行的 trimmed String”
   - `spawn_and_forget(cmd: &mut std::process::Command) -> Result<(), String>`：仅负责 `spawn()` 并把错误映射为 `String`（不做额外回退/吞错）

3) 为每个平台新增“只构建不执行”的构建函数，并让原 `run_command_*` 复用：
   - Windows（`#[cfg(target_os = \"windows\")]`）：
     - `build_command_windows(terminal_id: &str, command: &str) -> std::process::Command`
     - 约束：必须保留现有 argv 契约：
       - `wt`: `wt new-tab cmd /K <command>`
       - `cmd`: `cmd /K <command>`
       - `pwsh`: `pwsh -NoExit -Command <command>`
       - default(powershell): `powershell -NoExit -Command <command>`
   - macOS（`#[cfg(target_os = \"macos\")]`）：
     - `build_command_macos(terminal_id: &str, command: &str) -> std::process::Command`
     - 约束：保留现有转义逻辑（`\\` 与 `\"`）与脚本模板（Terminal vs iTerm）
   - Linux（`#[cfg(all(unix, not(target_os = \"macos\")))]`）：
     - `build_command_linux(terminal_id: &str, command: &str) -> std::process::Command`
     - 约束：保留现有 argv 结构（`bash -lc <command>` 且 `<command>` 为单参数；`gnome-terminal` 使用 `--` 分隔）

4) 改造 `run_command_windows/macos/linux`：
   - 只做：`let mut cmd = build_*...; spawn_and_forget(&mut cmd)?; Ok(())`
   - 禁止在这里新增“防御性回退”或静默降级（按 .ai/AGENTS.md 原则暴露失败）

5) 组织测试模块入口（避免把大量测试塞进 `terminal.rs`）：
   - 在 `terminal.rs` 末尾新增：
     - `#[cfg(test)] mod tests_exec;`
   - 计划在 `src-tauri/src/terminal/tests_exec.rs` 写单测（见 Task 2）
  </action>
  <verify>
1) `cargo test --manifest-path src-tauri/Cargo.toml`（本机当前 OS）  
2) `cargo check --manifest-path src-tauri/Cargo.toml`（确保无 unused/编译问题）
  </verify>
  <done>
- `terminal.rs` 中已存在可复用的 build 函数（不触发 spawn），且 `run_command_*` 复用它们。  
- `run_command_in_terminal()` 的 trim/拒绝行为未改变。  
- 单测入口 `tests_exec` 已连接（cargo test 能发现并执行）。  
  </done>

<task type="auto">
  <name>Task 2: 补齐“终端执行”单测（Windows 四分支 + macOS 转义 + Linux bash -lc 结构 + trim/拒绝 + spawn 失败传播）</name>
  <files>src-tauri/src/terminal/tests_exec.rs</files>
  <action>
新增 `src-tauri/src/terminal/tests_exec.rs`，只做“纯逻辑/命令构建层”的断言，禁止真实打开终端。

1) 公共测试工具（跨平台）：
   - 提供 helper 将 `Command` 的 program/args 统一转为可断言的 `Vec<String>`（使用 `to_string_lossy()`，避免 OsStr 平台差异）

2) 跨平台必测（不依赖 cfg）：
   - `sanitize_command()`：
     - 输入 `"   echo 1  "` → 返回 `"echo 1"`（锁定 trim 行为）
     - 输入 `\" \\n \\t \"` → 返回 Err（锁定全空白拒绝；只断言 `is_err()`）
   - `run_command_in_terminal()`：
     - 输入全空白 command → 必须返回 Err（只断言 Err，不锁定文案；且不应触发任何 spawn）
   - `spawn_and_forget()` 错误传播：
     - 构造 `Command::new(\"definitely-not-a-real-binary-xyz\")`，调用 `spawn_and_forget()` → 必须返回 Err（只断言 Err）

3) Windows（`#[cfg(target_os = \"windows\")]`）：
   - 对 `build_command_windows()` 分别覆盖 terminal_id：
     - `wt` / `cmd` / `pwsh` / `powershell`（默认分支可直接传入一个不匹配的 id，如 `\"powershell\"` 以外的值触发 default）
   - 对每个 case 精确断言：
     - program 名称
     - args 全量序列（顺序与每个参数内容都必须一致）

4) macOS（`#[cfg(target_os = \"macos\")]`）：
   - 对 `build_command_macos()`：
     - 覆盖 `iterm2` 与默认分支
     - 选取至少 1 个包含 `\\` 与 `\"` 的输入命令，精确断言最终传给 `osascript -e` 的脚本字符串（转义后的内容必须完全一致）

5) Linux（`#[cfg(all(unix, not(target_os = \"macos\")))]`）：
   - 对 `build_command_linux()` 覆盖 terminal_id：`gnome-terminal` / `konsole` / `alacritty` / default
   - 精确断言 argv 结构：
     - 必须包含 `bash`, `-lc`, `<command>`，且 `<command>` 为单参数（即 argv 序列中 `<command>` 仅出现一次）
     - `gnome-terminal` 分支必须包含 `--` 分隔

注意：
- 断言策略为“行为契约”，故允许精确断言 argv/脚本（这是锁定回归的目标）
- 禁止引入复杂 mock 框架来模拟 `std::process::Command`；只测 build 输出即可
  </action>

  </verify>
  <done>

- trim/拒绝与 spawn 失败传播路径覆盖完成，且测试不依赖真实终端环境。  
  </done>
