目标：不改变 tauri commands 的对外行为，仅增加“可注入依赖”的内部函数，避免单测改全局 env、并可稳定覆盖 `modified_ms=0` 回退分支。

1) home 解析可注入（避免测试 `set_var`）：
   - 新增内部函数：`resolve_home_dir_with(get_env: impl Fn(&str) -> Option<std::ffi::OsString>) -> Option<PathBuf>`
   - `resolve_home_dir()` 仍保留，内部改为调用 `resolve_home_dir_with(std::env::var_os)`（外部行为不变）
   - 逻辑必须保持与现状一致（Windows：USERPROFILE 优先，其次 HOMEDRIVE+HOMEPATH；非 Windows：HOME）

2) 目录路径拼接拆分为纯函数（便于断言契约）：
   - 新增内部函数：`resolve_user_commands_dir_path_in(home: &Path) -> PathBuf`
   - 规则固定：`home/.zapcmd/commands`

3) 目录创建提取为可复用函数：
   - 新增内部函数：`ensure_user_commands_dir_with(home: &Path) -> Result<PathBuf, String>`
   - `ensure_user_commands_dir()` 仍保留，内部改为：解析 home -> 调用 `ensure_user_commands_dir_with`
   - 错误信息需包含目标路径（便于用户自查；测试只断言关键片段，不锁死全文）

4) `modified_ms` 回退分支可测（metadata provider 注入）：
   - 新增内部函数：`read_single_user_command_file_with(path, read_to_string, metadata) -> Result<UserCommandFile, String>`
     - `read_to_string: impl Fn(&Path)->Result<String, std::io::Error>`
     - `metadata: impl Fn(&Path)->Result<std::fs::Metadata, std::io::Error>`
   - 生产函数 `read_single_user_command_file()` 调用 `read_single_user_command_file_with(path, fs::read_to_string, fs::metadata)`
   - 保持现有策略：metadata/modified 任一步失败 -> `modified_ms = 0`；读取 content 失败 -> Err

5) 单测入口：
   - 在文件末尾新增：`#[cfg(test)] mod tests_io;`

<task type="auto">
  <name>Task 2: 补齐 command_catalog 单测（IO 契约：路径/递归/排序/fail-fast/modifiedMs 回退）</name>
  <files>src-tauri/src/command_catalog/tests_io.rs</files>
  <action>
新增 `src-tauri/src/command_catalog/tests_io.rs`，覆盖以下“稳定契约”（按 05-CONTEXT.md 锁定）：

1) Windows home 解析回退顺序（`#[cfg(target_os = \"windows\")]`）：
   - 当 `USERPROFILE` 存在时优先返回它（即使 HOMEDRIVE/HOMEPATH 也存在）
   - 当 `USERPROFILE` 缺失时回退到 `HOMEDRIVE+HOMEPATH`
   - 当都缺失时返回 None（上层应报错）
   注意：测试必须通过 `resolve_home_dir_with(...)` 注入 env provider；禁止 `std::env::set_var`

2) 目录契约与创建：
   - `resolve_user_commands_dir_path_in(home)` 必须以 `.zapcmd/commands` 结尾
   - `ensure_user_commands_dir_with(home)` 在目录不存在时会创建目录（断言目录存在且为 dir）

3) 扫描规则（递归 + 过滤 + 排序）：
   - 在临时目录（自建 helper，使用 `std::env::temp_dir()` + 唯一子目录）下构造：
     - 顶层：`a.json`、`b.JSON`、`ignore.txt`
     - 子目录：`nested/c.json`
   - 调用 `collect_json_files_recursive(root)`：
     - 只返回 `.json/.JSON` 文件（大小写不敏感）
     - 递归包含子目录文件
     - 返回结果按实现规则“路径字符串排序”稳定（测试中用 `to_string_lossy()` 同规则生成 expected，避免写死分隔符/盘符）

4) 空目录：
   - 临时目录无任何 json 时：`collect_json_files_recursive` 返回空数组

5) fail-fast：
   - 对不存在的 root 路径调用 `collect_json_files_recursive` 必须返回 Err
   - 仅断言错误信息包含“目标路径”关键片段（不锁定系统错误全文）

6) `modified_ms` 回退 0：
   - 直接调用 `read_single_user_command_file_with`，让 `read_to_string` 返回固定内容、`metadata` 返回 Err
   - 断言返回的 `modified_ms == 0` 且 `content` 透传

注意：
- Rust 层不做 JSON 合法性校验：不要在 Rust 测试里尝试 parse JSON。
- 尽量避免新增 dev-dependency（如确需新增，必须在 action 中写清理由与影响）。
