   - `struct MonitorInfo { name: Option<String>, position: PhysicalPosition<i32>, size: PhysicalSize<u32> }`
   - 由 production 代码把 Tauri `Monitor` 映射为 `MonitorInfo`（不改变外部行为）

2) 提取 restore 位置选择逻辑为纯函数（可单测）：
   - 新增内部函数（命名可调整，但职责必须明确）：\n
     `resolve_restored_window_position_with(saved: &MainWindowBounds, window_size: PhysicalSize<u32>, monitors: &[MonitorInfo], primary: Option<&MonitorInfo>) -> PhysicalPosition<i32>`
   - 逻辑必须与现状一致：
     - monitors 为空：返回 desired（不做 clamp）
     - display_name 命中：选该 monitor 并 clamp
     - desired 落点命中：选落点 monitor 并 clamp
     - 否则：primary -> first 居中并 clamp

3) 提取“随鼠标屏重定位”决策为纯函数（提供拒绝路径）：
   - 新增内部函数：\n
     `compute_reposition_to_cursor_monitor(cursor: PhysicalPosition<i32>, window_pos: PhysicalPosition<i32>, window_size: Option<PhysicalSize<u32>>, monitors: &[MonitorInfo]) -> Option<PhysicalPosition<i32>>`
   - 逻辑必须与现状一致：
     - monitors 为空 -> None
     - cursor 不在任何 monitor -> None（拒绝路径）
     - cursor 与 window 在同一 monitor -> None（不动作）
     - cursor 在某 monitor 且 window 不在同 monitor -> Some(center_in_monitor(...))
     - window_size 为 None 时使用与现状一致的默认值（680x124）
   - 现有 `reposition_to_cursor_monitor(window)` 保持对外行为：只薄封装读取 cursor/window/monitors + set_position
