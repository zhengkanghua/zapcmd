  <name>Task 1: 在 precommit-guard 中实现 staged 文件分类与 coverage 触发规则</name>
  <files>scripts/precommit-guard.mjs</files>
  <action>
实现“快路径 + 条件触发 coverage”的双通道门禁，严格遵守 CONTEXT.md 的触发规则与例外：

1) 仍使用 git staged 列表作为唯一输入（`git diff --cached --name-only --diff-filter=ACMR`），确保路径按 `/` 统一处理。  
2) 增加“纯文档/说明类改动直接跳过（不跑任何门禁）”逻辑：
   - `docs/**`（但排除 `docs/command_sources/_*.md`）
   - `README*`、`CHANGELOG.md`
   - 仅 `.github/workflows/**`
3) 执行快路径（保持当前顺序与行为）：
   - `npm run lint`
   - `npm run typecheck`
   - 若存在 `src/` 的 `.ts/.vue` 变更（排除 `__tests__`、`*.test.ts`、`*.spec.ts`）→ `npm run test:related -- <targets>`
   - 若命中测试/类型相关文件（`__tests__`、`*.test.ts`、`*.spec.ts`、`vitest.config.ts`、`tsconfig.test.json`）→ `npm run typecheck:test`
   - 若命中 Rust 变更（`src-tauri/**`/`.rs`/Cargo*）→ `cargo check --manifest-path src-tauri/Cargo.toml`
4) 计算 `shouldRunCoverage`，并在命中时追加执行 `npm run test:coverage`：
   - 关键配置：`package.json`、`package-lock.json`、`vitest.config.ts`、`tsconfig*.json`、`scripts/**`、`.githooks/**`、`src-tauri/tauri.conf.json`、`src-tauri/capabilities/**`、`.env.keys`、`Cargo.lock`
   - 高风险 src 子集：`src/features/security/**`、`src/services/commandExecutor.ts`、`src/services/updateService.ts`、`src/services/tauriBridge.ts`
   - 高风险 Rust 文件：`src-tauri/src/terminal.rs`、`src-tauri/src/command_catalog.rs`、`src-tauri/src/bounds.rs`
   - 运行时资产：`assets/runtime_templates/commands/**`（排除 README/说明）
   - 大改动阈值：staged 的 `src/` 业务代码文件数（`.ts/.vue`，排除 tests）> 20
   - 排除：仅测试文件改动、仅样式改动（但仍跑快路径）

注意：不提供任何 bypass 开关（env 或提交标记）。
  </action>
  <verify>
手动验证（建议分 3 组快速自检，每组只做最小 staged 改动，不需要真实提交）：
1) 仅改 `README.md` → 运行 `npm run precommit:guard` 应直接退出（不打印 lint/typecheck）。  
2) 改 `src/services/updateService.ts` 或 `package.json` → 应明确打印触发原因并执行 `npm run test:coverage`。  
3) 改 `src/features/commands/__tests__/...` → 不应触发 `test:coverage`，但应执行 `typecheck:test`。
  </verify>
  <done>
- doc-only 变更时不再运行任何门禁命令，脚本直接返回 0。  
- 命中触发规则时会追加执行 `npm run test:coverage`，失败阻止提交。  
- 触发 coverage 时的输出包含：原因/命中文件/命令清单，信息可读。
  </done>
