<task type="auto">
  <name>Task 1: 增加桌面端 E2E 冒烟脚本与 npm scripts（Windows）</name>
  <files>scripts/e2e/desktop-smoke.cjs, package.json, .gitignore</files>
  <action>
目标：提供一个“单命令可运行”的 E2E 冒烟入口，并且失败时可产出截图/日志用于定位。

实现建议：
1) 新增 `scripts/e2e/desktop-smoke.cjs`（CommonJS，避免 ESM 与 WebDriver 依赖的互操作坑），负责：
   - 校验运行平台（仅 Windows 支持；非 Windows 直接报错并退出非 0，避免误把 skip 当 pass）
   - 校验依赖存在（`tauri-driver` 可执行、`msedgedriver` 可用），缺失时打印“如何安装/如何修复”的明确指令
   - 以子进程启动 `tauri-driver`（固定端口 4444，写入日志文件），等待 `/status` 可用后再继续
   - 以 WebDriver client（建议 `selenium-webdriver`）启动应用并执行 1 条冒烟用例：
     a) 等待 `#zapcmd-search-input` 可见  
     b) 输入确定有结果的关键词（如 `git`）  
     c) 断言 `[aria-label=\"result-drawer\"]` 出现且包含至少 1 个 `.result-item`  
     d) 发送 `Escape`，断言抽屉关闭且 input 清空
   - 失败时：保存截图（PNG/Base64 解码）+ 关键日志（driver/stdout/stderr）到 `.tmp/e2e/desktop-smoke/`
   - 清理：无论成功失败都要关闭 driver 并 kill `tauri-driver` 子进程（避免 CI 卡死）
2) `package.json` 新增 scripts（命名固定，便于 CI 调用）：
   - `tauri:build:debug`：`tauri build --debug --no-bundle`（可复用现有 `version:sync`/`keys:sync` 前置）
   - `e2e:desktop:smoke`：调用上述脚本（内部可按需先 build 再跑；或允许通过 env 指定已构建好的 app path）
3) `.gitignore` 增加 `.tmp/`（避免本地生成的 E2E 产物污染 git status）

注意：E2E 仅做“启动 + 搜索 + 抽屉开合”，不做真实终端执行。
  </action>
  <verify>
Windows 本地验证（一次即可）：
1) `npm ci`  
2) `npm run tauri:build:debug`（生成 debug 可执行文件）  
3) `npm run e2e:desktop:smoke`  
预期：脚本成功退出 0；失败时 `.tmp/e2e/desktop-smoke/` 下出现截图/日志。
  </verify>
  <done>
- 存在可运行的 `npm run e2e:desktop:smoke`，能完成“启动+搜索+抽屉开合”冒烟。  
- 失败会产出截图与日志到 `.tmp/e2e/desktop-smoke/`。  
- 脚本在成功/失败后都能正确清理子进程，不会 hang。
  </done>
