<task type="auto">
  <name>Task 1: 在 CI Gate Windows quality-gate 中加入“内置命令生成一致性”阻断检查</name>
  <files>.github/workflows/ci-gate.yml</files>
  <action>
在 `jobs.quality-gate`（Windows）中，`npm run check:all` 之前新增一个明确命名的 step：
1) 运行生成脚本：`pwsh -File scripts/generate_builtin_commands.ps1`  
2) 校验产物是否已提交：`git diff --exit-code -- assets/runtime_templates/commands/builtin docs/builtin_commands.generated.md`

要求：
- diff 非空 → step 直接失败，并在输出中打印“如何修复”的指令（建议 echo 一段说明）
- step 名称必须清晰（日志一眼定位）
- 本 step 仅在 Windows gate 执行（保持其它 OS smoke 不变）
  </action>
  <verify>
在本地故意制造不一致（只改 `docs/command_sources/_git.md` 不跑生成），推到分支触发 CI，预期 Windows quality-gate 在该 step 失败并提示修复命令。
  </verify>
  <done>
- Windows quality-gate 在 `check:all` 前会检查生成产物一致性。  
- 失败日志包含明确修复指令（可复制粘贴执行）。
  </done>
</task>

<task type="auto">
  <name>Task 2: 新增独立的 Windows 桌面端 E2E 冒烟 job，并上传截图/日志</name>
  <files>.github/workflows/ci-gate.yml</files>
  <action>
新增一个独立 job（例如 `desktop-e2e-smoke`），要求：
- runs-on: windows-latest
- 必须阻断（job 失败即 PR 不可合并）
- 不重试（不要加 retry 机制）
- 失败诊断产物：上传 `.tmp/e2e/desktop-smoke/` 目录（`actions/upload-artifact@v4`，`if: always()`）
- Node/Rust 环境：与现有 quality-gate 保持一致（Node 20 + Rust stable + npm ci）
- 执行顺序（建议）：
  1) 安装 WebDriver 依赖：`cargo install tauri-driver --locked`、`pwsh -File scripts/e2e/install-msedgedriver.ps1`
  2) 构建 debug app：`npm run tauri:build:debug`（或由 `npm run e2e:desktop:smoke` 内部触发）
  3) 执行 E2E：`npm run e2e:desktop:smoke`

注意：该 job 只跑最小冒烟，不跑 `check:all`（避免重复耗时）。
  </action>
  <verify>
在 CI 上制造一个可控失败（例如把 E2E 选择器暂时改错），确认：
1) job 失败且阻断  
2) artifact 中包含截图与日志  
3) 恢复后 job 通过
  </verify>
  <done>
- CI Gate 中存在独立的 Windows E2E job，失败阻断合并。  
- 无论成功/失败，都会上传 `.tmp/e2e/desktop-smoke/` 作为定位材料。
  </done>
</task>

  </done>
