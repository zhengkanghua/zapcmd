  <action>
新增一个 Node 脚本，用于从 `coverage/lcov.info`（优先）与 `coverage/index.html`（可选）读取数据并打印：

1) **四项覆盖率总览**（Statements/Branches/Functions/Lines）：当前值 + 90% 门槛
2) **Top deficits（缺失分支）**：按缺失分支数降序列出 Top 10–20（显示：文件、branches%、缺失分支数）
3) **Top deficits（缺失行）**（可选）：按缺失行数降序列出 Top 10–20
4) **指引**：提示 HTML 报告入口 `coverage/index.html`

要求：
- 输出使用中文标题（日志中可以中英混用），便于 CI 直接阅读
- 读取失败时要给出清晰错误提示（例如 lcov 不存在时提示先运行 `npm run test:coverage`）
  </action>
  <verify>
在本地运行一次 `npm run test:coverage`，确认覆盖率输出中能看到新增的“总览 + Top deficits + HTML 指引”。
  </verify>

- `coverage-report.mjs` 可独立运行并输出可定位信息（无需打开 HTML 也能知道优先补哪些文件/分支）。
  </done>
