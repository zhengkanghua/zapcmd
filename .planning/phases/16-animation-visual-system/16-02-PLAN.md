<task type="auto">
  <name>Task 1: Review overlay 开合动效落地（dim→panel / panel→dim），并遵循“先稳定尺寸再动画”</name>
  <files>src/styles.css</files>
  <action>
在不引入动画库的前提下，基于 `review-overlay--${stagingDrawerState}` 实现动效：\n
1) 为 `.review-overlay--opening` / `--closing` 增加 scrim 与 panel 的动效规则：\n
   - opening：scrim（dim）先渐显；panel 再从右侧轻位移滑入（可使用 20~40ms 的 delay 表达“dim 先出现”）；\n
   - closing：panel 先滑出；scrim 再渐隐（可使用 delay）；\n
   - 总时长控制在 `180~220ms`，目标约 `200ms`；只动 `opacity/transform`，禁止用影响布局的属性做动画。\n
2) 视觉约束：\n
   - dim alpha 参考 `docs/ui-redesign/11-b4-visual-spec.md`（约 0.28~0.36），保持低噪音；\n
   - Review 面板层级应更“实”（更强 shadow/border），符合“专业桌面工具面板”基线。\n
3) “先稳定尺寸再动画”策略：\n
   - overlay 仍在 `opening` 状态挂载，但 panel 动画建议带少量 delay，给 window resize 留出先手；\n
   - 禁止在 opening/closing 期间触发重复 resize（后续 Task 2 会对齐计时常量）。\n
  </action>
  <verify>
手动 UI smoke（Windows 优先）：\n
- Review 打开：dim 先出现再滑入；关闭：先滑出再去 dim；整体约 200ms；\n
- opening/closing 期间窗口边界无肉眼可见反复跳动/闪烁（若出现，必须回到“一次性 resize + 内部动画”策略，禁止动画期间反复 resize）。\n
  </verify>
  <done>
Review overlay 动效满足时序与克制要求；并且在 Windows 下开合期间 resize 稳定可用。
  </done>
</task>
