## Critical Gaps

### Requirement Gaps

- **COV-01 / COV-02** — Phase 2 has completed plans and summaries, but `.planning/phases/02-coverage-gate-90/02-VERIFICATION.md` does not exist, so both requirements are orphaned from formal milestone verification.
- **UX-01 / UX-02** — Phase 9 also lacks `.planning/phases/09-ui-ux-polish/09-VERIFICATION.md`, and its summaries do not include `requirements-completed` frontmatter, leaving both UX requirements orphaned.
- **E2E-02 (partial)** — The requirement is still only partially delivered, and the Phase 10 verification overstates the current state of macOS local/CI gating.

### Cross-Phase Integration Issues

- **Phase 10 docs → `scripts/verify-local-gate.mjs`:** Phase 10 planning, summaries, and verification say macOS desktop smoke runs by default in `npm run verify:local`, but the implementation keeps macOS behind `--macos-desktop-e2e-experimental` or `ZAPCMD_E2E_EXPERIMENTAL_MACOS=1`.
- **Phase 10 docs → GitHub workflows:** Phase 10 artifacts say CI/release contain blocking macOS desktop smoke jobs, but `.github/workflows/ci-gate.yml` and `.github/workflows/release-build.yml` only wire Windows desktop smoke plus generic macOS build/test smoke.

### Broken Flows

- **macOS local verification flow:** breaks at `npm run verify:local`, because desktop smoke is skipped by default on macOS.
- **macOS remote gate flow:** breaks in GitHub Actions, because there is no `desktop-e2e-smoke-macos` job despite the Phase 10 docs claiming one exists.

## Tech Debt / Deferred Items

- Linux full-matrix desktop E2E remains deferred beyond v1.0 in `.planning/REQUIREMENTS.md`.
- macOS desktop smoke remains operationally risky because SafariDriver / WKWebView stability needs continuous observation.
- `docs/active_context.md` contains contradictory statements about the default macOS `verify:local` behavior and should be reconciled after the implementation decision is made.

## Recommendation

Do **not** archive the milestone yet. The safest next step is to close the audit gaps first:

1. Add `02-VERIFICATION.md` for Phase 2 and `09-VERIFICATION.md` for Phase 9.
2. Decide the real Phase 10 contract for macOS desktop smoke:
   - either enable it by default locally and add real macOS desktop-smoke CI/release jobs,
   - or downgrade the docs/verification artifacts so they match the current experimental-only behavior.
3. Re-run `$gsd-complete-milestone` after those gaps are resolved.
